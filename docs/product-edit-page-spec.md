# 产品编辑类二级页面设计规范

## 1. 适用范围

本规范适用于从一级列表进入、承载较多表单编辑和阶段化配置的后台二级页面，包括但不限于：

- 旅游线路编辑、新建线路
- 研学线路编辑
- 自由行套餐编辑
- 单项服务编辑
- 外采产品包装编辑
- 团期、航次、班期编辑
- 方案、费用说明、素材、价格库存等配置类页面

不适用：

- 一级列表页
- 纯查看详情页，详情页遵守 `docs/secondary-page-ui-spec.md`
- 财务单据处理页
- 报表、看板、工作台
- 轻量弹窗或右抽屉表单

## 2. 总体原则

产品编辑类二级页面的目标是让业务人员在一个稳定、可预期的编辑工作台中完成对象维护。

必须遵守：

- 页面不展示产品建模解释、字段分层说明、数据链路说明或内部口径说明。
- 页面只展示当前角色需要填写、核对、调整和提交的真实业务字段。
- 页面画布可以 100% 宽；步骤 Tab 铺满页面内容宽度，Tab 下方核心编辑内容必须有最大宽度，避免宽屏拉散。
- 同类编辑页必须共享页面骨架、摘要卡、步骤 Tab、表单卡、素材上传、明细卡和底部步骤按钮样式。
- 业务差异只体现在字段、明细结构和状态动作上，不重新设计页面外观。

## 3. 页面骨架

标准结构：

```html
<main class="edit-page product-edit-page">
  <section class="page-workbar edit-workbar">...</section>
  <section class="edit-summary-card">...</section>
  <nav class="edit-step-tabs">...</nav>
  <section class="edit-workspace">
    <section class="edit-panel active">
      <section class="edit-form-card">...</section>
    </section>
  </section>
</main>
```

当前线路编辑页已经形成的实现类可作为第一版参考：

- `route-blue-edit-page` 对应 `edit-page`
- `route-blue-workbar` 对应 `edit-workbar`
- `route-blue-summary` 对应 `edit-summary-card`
- `route-blue-progress` 对应 `edit-step-tabs`
- `route-blue-card` 对应 `edit-form-card`
- `route-media-*` 对应 `edit-media-*`
- `route-itinerary-*` 对应 `edit-detail-*`

后续新增页面应优先使用通用 `edit-*` 类；现有 `route-blue-*` 类逐步迁移为兼容别名。

## 4. 宽度与对齐

页面规则：

- 页面画布宽度：`100%`
- 顶部工作栏、摘要卡和步骤 Tab：`100%` 铺满当前后台内容区。
- Tab 下方核心编辑内容最大宽度：`1216px`
- 核心编辑内容左对齐，不居中漂浮。
- 表单卡内部字段、素材区、明细卡和底部步骤按钮必须遵守同一内容宽度逻辑。
- 宽屏时不允许把输入框、卡片、指标区无限拉宽。

推荐：

```css
.edit-content-width {
  width: 100%;
  max-width: 1216px;
}
```

## 5. 顶部工作栏

工作栏用于页面级操作，结构固定：

- 左侧：返回入口 + 当前页面标题
- 右侧：页面级主操作按钮组

规则：

- 返回文案统一使用 `返回`，不拼接上级页面名称。
- 不放面包屑。
- 不放解释性副标题。
- 页面级主操作只出现一次，统一在工作栏右侧。
- 摘要卡、表单卡、明细卡内不得重复放同一组全局主操作。

示例按钮：

- `保存草稿`
- `提交审核`
- `开排团期`
- `取消`

## 6. 摘要卡

摘要卡用于让用户确认当前编辑对象，不承担编辑表单职责。

推荐内容：

- 编号或业务识别信息，如线路编号、团期编号、航次编号、班期编号
- 状态标签
- 产品类型或业务类型
- 最后编辑时间
- 主标题
- 副标题或一句业务摘要
- 标签
- 关键属性，如供货方、承接组织、出发地
- 2 到 3 个轻量指标，如方案总数、参考起价、数据完整度

禁止：

- 大段说明文字
- 对象链路说明
- 字段口径说明
- 与工作栏重复的主操作按钮
- 无业务含义装饰图标

视觉规则：

- 顶部可使用 4-5px 主色细条。
- 卡片圆角不超过 8px。
- 指标卡使用细边框和浅底，不做大面积色块。
- 底部上下留白一致，避免视觉挤压。

## 7. 步骤 Tab

步骤 Tab 用于阶段化编辑，不用于状态筛选。

规则：

- 使用数字圆点 + 文案。
- 选中态使用浅蓝底和主色文字。
- 不使用底部粗下划线。
- 不因 active 导致高度、字号、布局跳动。
- Tab 数量建议 3-5 个。
- 步骤 Tab 自身铺满页面内容区，不能被下方 `1216px` 内容轨道限制。
- 移动端或窄屏允许横向滚动。

常见步骤：

- 基础信息
- 行程方案
- 费用说明
- 价格库存
- 素材附件

## 8. 表单卡与小模块

一个 Tab 内尽量使用一个整体表单卡，不把每个小模块拆成多个漂浮卡片。

表单卡规则：

- 卡片圆角不超过 8px。
- 内边距采用 18-20px。
- 默认边框、分割线、输入框普通态边框使用中性灰，不使用带明显蓝感的边线。
- 主色只用于当前步骤、主按钮、链接、选中态和输入框聚焦态。
- 输入框高度使用 32px。
- 表单网格默认两列或三列，复杂字段横跨整行。
- 小模块之间用小标题区分。
- 分隔线放在小标题下方，不放在模块之间形成多余横线。

小标题规则：

- 标题左侧使用 4px 主色短竖条。
- 标题字号 15px 左右。
- 小标题下面可以有细分隔线。
- 不写解释性说明，除非它是当前操作必要提示。

## 9. 素材与附件上传区

素材上传区必须同时考虑上传前、上传后、多图和附件列表状态。

标准结构：

- 标题图片：大预览图 + 文件名/尺寸/大小 + 替换 + 删除
- 海报：多张缩略图 + 上传入口 + 排序 + 批量删除
- 行程附件：文件列表 + 文件类型/大小 + 删除 + 上传附件

视觉规则：

- 已上传图片必须显示真实图片，不使用颜色块或渐变占位。
- 图片信息不使用白底大块压在图片上，推荐使用底部渐变 + 白字。
- 删除类操作默认使用灰色垃圾桶图标，hover 时才显示红色危险反馈。
- 替换、排序等常规操作使用图标 + 简短文字。
- 上传入口使用浅蓝底或白底虚线按钮，图标 + 简短文字。
- 图片卡、附件卡不出现说明性长文案，只展示上传状态、文件信息和操作。

推荐操作命名：

- `替换`
- `排序`
- `上传海报`
- `上传附件`

不推荐：

- `替换图片`
- `调整顺序`
- `点击这里上传用于渠道素材的多张宣传海报`

## 10. 明细编辑卡

用于分日行程、费用项、资源项、价格库存项、乘客结构项等重复明细。

标准结构：

- 左侧稳定标识，如 D1、费用项、舱型、铺位、套餐项
- 中间主内容编辑区
- 右侧辅助字段或摘要字段
- 明细卡之间保持 12-16px 间距
- 明细卡内字段不应过度拆碎

行程类明细推荐：

- 日期编号：D1/D2/D3
- 标题输入
- 行程正文输入
- 餐饮/住宿安排输入
- 常用选项按钮

## 11. 按钮规则

页面级按钮：

- 放在顶部工作栏右侧。
- 主操作使用 `.btn-primary`。
- 次操作使用 `.btn-secondary`。
- 不在摘要卡和表单卡内重复出现。

步骤按钮：

- 放在当前 Tab 表单卡底部。
- 左侧为上一步，右侧为下一步或提交。
- 和表单内容保持同一最大宽度。

素材工具按钮：

- 常规动作：图标 + 简短文字。
- 危险动作：图标按钮，默认灰色，hover 红色。
- 上传动作：图标 + 简短文字，浅色虚线按钮。

## 12. CSS 命名建议

后续应逐步沉淀通用类，避免每个业务页面创建一套视觉类。

推荐通用类：

- `edit-page`
- `edit-workbar`
- `edit-summary-card`
- `edit-step-tabs`
- `edit-workspace`
- `edit-panel`
- `edit-form-card`
- `edit-subsection`
- `edit-subsection-titlebar`
- `edit-media-grid`
- `edit-media-panel`
- `edit-media-preview`
- `edit-media-actions`
- `edit-media-action-btn`
- `edit-media-icon-btn`
- `edit-detail-card`
- `edit-tab-actions`

迁移策略：

1. 当前 `route-blue-*` 样式保留，作为旅游线路编辑页稳定实现。
2. 新增通用 `edit-*` 类，与 `route-blue-*` 共用同一 CSS 规则。
3. 后续编辑页优先使用 `edit-*` 类。
4. 当所有编辑页迁移完成后，再减少业务专属视觉类。

## 13. 验收清单

每个产品编辑类二级页面完成后必须检查：

- 是否有明确返回入口和页面标题。
- 返回入口是否只显示 `返回`。
- 页面级主操作是否只出现一次。
- 步骤 Tab 是否 100% 铺满，Tab 下核心内容是否限制在 1216px 左右的稳定宽度内。
- 摘要卡是否只展示对象摘要、状态、关键属性和轻量指标。
- 步骤 Tab 是否无下划线、无跳动、可切换。
- 一个 Tab 内是否是整体表单，不是多个散卡堆叠。
- 小模块分隔线是否只在小标题下方。
- 素材区是否覆盖上传后、多图、附件列表和删除/替换/排序动作。
- 删除图标是否默认灰色，hover 才红色。
- 是否没有说明性建模语言、字段口径说明、对象链路说明。
- 默认线条是否为中性灰，蓝色是否只用于真实交互状态。
- 是否没有内联 `style` 或页面内 `<style>`。
- 是否通过 `git diff --check`。
