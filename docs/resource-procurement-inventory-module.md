# 交通资源采购模块

更新时间：2026-07-06

## 1. 模块定位

交通资源采购是自营产品的动态供给台账，主入口放在“履约中心 / 交通采购”，下设“机票切位、邮轮包舱、专列包铺”。它维护具体日期、具体数量、具体成本、分期付款和释放损失规则，不属于资源模块的基础档案，也不再作为团期管控下的普通子页。

资源模块继续维护低频静态资料：

- 航空公司、航线库、航司协议
- 船公司、船只档案、邮轮模板，船方协议规则并入船公司详情
- 专列运营商、车次库、专列模板，专列协议规则并入运营商详情
- 供应商档案

交通资源采购维护高频动态对象：

- 交通资源采购单
- 机票位采购库存
- 邮轮/游轮舱房采购库存
- 专列铺位采购库存
- 分期付款计划
- 可损规则
- 团期借调记录
- 库存流水

## 2. 使用角色

| 角色 | 高频任务 | 下游影响 |
|---|---|---|
| 采购部 | 采购专列铺位、邮轮舱房等大资源 | 形成可借调库存和成本基准 |
| 航服 | 采购和维护机票团队位、PNR、出票节点 | 影响出境/国内团期可开售人数 |
| 资源调度 | 维护舱型、铺位和多批库存调配 | 影响团期借调和释放 |
| 计调 | 开团时借调库存、关注可用和可释放 | 影响团期可售容量 |
| 财务 | 查看付款节点承接付款执行 | 不在本模块执行真实付款 |

## 3. 业务流程

```text
新增采购 -> 采购入库 -> 团期借调 -> 订单成交扣减 -> 团期关闭归还/释放 -> 批次关闭
```

流程规则：

1. 自营国内游、出境游如提前采购机票位，团期开售前必须从机票位采购库存借调人数。
2. 自营邮轮航次必须按航次和舱型从邮轮舱房库存借调。
3. 自营专列班期必须按班次、席别和铺位从专列铺位库存借调。
4. 借调数量不能超过可用库存。
5. 团期出行日期、返程日期必须承接所选采购库存日期。
6. 订单成交由销售中心自动消耗团期已借调库存，并同步反映到采购库存已售数。
7. 团期关闭时，未售出的已借调数量归还采购库存或标记释放。
8. 外采产品不做硬性采购库存拦截，但应提示超卖和滞销风险。
9. 交通资源采购页不提供常规手工成交扣减入口，只查看成交消耗结果和库存流水。
10. 历史补录、扣减失败、释放修正、报损修正等异常场景通过详情页“库存调整”处理，并必须记录原因和流水。

## 4. 新增采购录入

新增入口只服务采购人员快速形成动态库存，不在首次录入时塞入供应商、协议、币种等基础配置字段。采购类型由三级菜单自动决定，不显示采购模式、合同号或供应商输入框；机票选择出发/回程航班，邮轮选择出发/返港航次，专列选择去程/返程车次，系统从资源配置自动识别供应商、协议规则、币种和可损释放口径。

| 字段 | 录入方式 |
|---|---|
| 采购类型 | 由三级菜单自动决定，机票切位、邮轮包舱、专列包铺不在抽屉里再手工选择 |
| 资源选择 | 必填，机票选出发/回程航班，邮轮选船只/航次，专列选出发/返程车次 |
| 系统识别 | 自动展示供应商、协议规则、币种和释放规则，只读展示 |
| 类型明细 | 必填，机票维护航班/日期/舱位/座位数，邮轮维护船只/航次/舱型/舱房床位，专列维护班次/车厢/席别/铺位 |
| 押金阶段 | 默认一压、二压、三压/尾款，可新增阶段；每个阶段可增加增减数量行 |
| 总成本 | 系统按明细数量和单价自动汇总 |

采购新增不提供供应商、协议、币种手填入口。若航班、航次或车次不存在，应先回资源模块补充航空公司/航班、船只/航次模板或车次库；采购单保存后记录系统识别快照，用于列表、详情、付款节点和后续成本承接。

以下字段不在新增抽屉中人工填写：

- 合同号/协议号：系统生成。
- 采购角色：按采购对象带出，机票默认航服，邮轮默认资源调度，专列默认采购部。
- 供应商、协议、币种：由资源配置自动识别并保存快照。
- 供应商确认状态：新增后默认为待确认，由后续确认动作维护。
- 付款状态：由付款节点和财务承接动作生成。
- 已借调、已售、可用、已释放、库存流水：由库存动作自动计算。

## 5. 采购单记录字段

| 字段 | 要求 |
|---|---|
| 采购单号 | 自动生成 |
| 采购对象 | 机票位、邮轮/游轮舱房、专列铺位 |
| 采购角色 | 采购部、航服、资源调度、计调，新增时按对象默认带出 |
| 供应商 | 由航班/航次/车次识别出的结算供应商 |
| 协议来源 | 由航司协议、船公司规则、运营商规则识别 |
| 出行日期 | 来源于采购明细 |
| 返程日期 | 来源于采购明细 |
| 采购数量 | 来源于采购明细汇总 |
| 单价 | 来源于采购明细，采购单层展示平均单价或主单价 |
| 总成本 | 自动计算 |
| 币种 | CNY、EUR、USD 等 |
| 供应商确认状态 | 待确认、已确认、部分确认、变更中、已取消 |
| 采购状态 | 草稿、待确认、已入库、部分借调、已关闭 |

## 6. 押金/付款阶段

每张采购合同默认维护一压、二压、三压/尾款三个节点，也支持新增节点。新增时只录入业务付款计划和提醒口径，不自动生成付款申请，不自动进入审批流：

| 字段 | 说明 |
|---|---|
| 付款期次 | 第一次付款、第二次付款、第三次付款 |
| 锁定数量 | 当前节点锁定的位置数量 |
| 付款比例 | 当前节点付款比例 |
| 单价 | 当前节点对应采购单价 |
| 付款日期 | 计划付款日期 |
| 增减数量 | 节点内可新增增减数量行，支持正数增加和负数减少 |
| 应付金额 | 按锁定数量、单价和付款比例自动计算 |
| 提醒状态 | 到期前提醒经办人核实数量和金额；经办人确认后手动发起付款申请 |

付款计划只表达业务侧付款节点和状态，真实付款申请、付款执行、回单和 NC 推送仍归财务模块。

## 7. 可损规则

可损规则从押金/付款阶段承接，可在详情中复核和调整：

| 字段 | 说明 |
|---|---|
| 关联节点 | 第一次付款前、第二次付款前、第三次付款前 |
| 无损释放数量 | 该节点前可无损释放或调整的位置数量 |
| 无损释放比例 | 该节点前可无损释放比例 |
| 最晚释放日 | 节点前最后释放日期 |
| 节点后损失规则 | 付款后释放的损失口径 |

## 8. 数量口径

| 字段 | 口径 |
|---|---|
| 总采购数 | 采购单形成的原始库存数量 |
| 可用数 | 总采购数 - 已借调数 - 已报损数 |
| 已借调数 | 已分配给团期且尚未归还的数量 |
| 已售数 | 已借调库存中被订单成交消耗的数量 |
| 可释放数 | 已借调数 - 已售数 |
| 已释放数 | 团期关闭或节点释放回采购库存的数量 |
| 已报损数 | 超过释放节点、取消损失或人工确认损失的数量 |

## 9. 采购对象差异

### 9.1 机票位

字段：

- 出发航班、回程航班、出发日期、回程日期
- 舱位、团队 PNR、采购位数
- 名单截止日、出票截止日

规则：

- 同一日期、同一航班允许多批采购库存。
- 开团时可从一批或多批机票库存借调。
- 团期日期必须与所选机票库存日期一致。
- 借调人数不得超过机票可用库存。
- 团期关闭后未售机票位归还原库存或标记释放。

### 9.2 邮轮/游轮舱房

字段：

- 船公司、船只、航次
- 出发港、返程港、启航日期、返程日期
- 舱型：内舱、海景、阳台、套房
- 舱房数量、床位数量

规则：

- 邮轮航次开团必须从对应航次、对应舱型借调库存。
- 各舱型借调数量不得超过对应舱型可用库存。
- 价格矩阵按舱型、成人/儿童、占位/不占位维护。
- 儿童占位按对应舱型计价；儿童不占位使用单独价格。
- 第三/第四人单独维护价格规则。

### 9.3 专列铺位

字段：

- 运营商、出发车次、返程车次
- 出发日期、返程日期、车厢
- 席别：软卧、硬卧
- 铺位：上铺、中铺、下铺

规则：

- 专列班期开团必须从对应班次、席别、铺位借调库存。
- 各铺位类型借调数量不得超过可用库存。
- 价格矩阵按软卧/硬卧、上中下铺、成人/儿童、占位/不占位维护。
- 儿童占位按具体铺位计价；儿童不占位使用单独价格。

## 10. 页面结构

### 10.1 交通资源采购列表

列表字段：

- 出发资源：机票显示出发航班，邮轮显示出发航次，专列显示去程车次
- 回程资源：机票显示回程航班，邮轮显示返港航次，专列显示返程车次；单程未采购时显示未采购
- 出发日期
- 回程日期：显示回程资源的出发日期；单程未采购时不硬填日期
- 用量：已用/总量、进度条、剩余量
- 待办事项：付款核实、DL还位核对、损耗确认、名单确认；每行只展示当前最需要处理的一类任务
- 操作

行操作：

- 详情
- 借调到团期

列表不再把合同号、资源类型、采购模式、关联协议号和复杂风险状态作为首屏主信息。交通资源采购页的首要任务是让采购、航服、资源调度一眼判断采购的是哪趟航班/航次/车次、对应日期、库存使用压力以及当前最该处理的待办事项。

### 10.3 详情抽屉

详情分区：

- 采购单信息
- 库存明细
- 分期付款计划
- 可损规则
- 团期借调记录
- 库存流水

详情操作：

- 借调到团期：将采购库存分配给团期。
- 关闭采购单：团期关闭或批次收尾时释放未售库存。
- 补充到资源档案：当采购过程中发现航班、航次、车次或规则缺失时，回到资源模块补充基础配置。
- 库存调整：仅用于异常修正，不替代销售中心的正常成交扣减。

### 10.4 团期开团页

新建团期页必须表达：

- 借调库存来源
- 借调数量
- 借调状态
- 日期承接
- 资源校验结果

提交校验：

- 自营国内游、出境游、邮轮、专列必须完成资源借调。
- 借调数量必须覆盖可售容量。
- 团期出行日期和返程日期必须与所选库存一致。

### 10.5 订单成交扣减

订单成交扣减不在交通资源采购页手工执行。

- 销售在销售中心下单并确认成交。
- 系统自动扣减团期已借调库存。
- 采购库存同步增加已售数。
- 库存流水记录“订单成交扣减”。
- 交通资源采购页只展示成交消耗结果，不提供常规成交扣减按钮。

### 10.6 异常库存调整

异常库存调整只用于：

- 历史订单补扣。
- 线下成交补录。
- 订单库存扣减失败修正。
- 团期关闭释放修正。
- 报损或盘点修正。

调整时必须填写库存明细、调整类型、数量、关联团期/订单和原因，并形成“人工调整”流水。

## 11. 验收清单

阶段二验收必须能表达：

- 新增采购轻量录入，不手填供应商、协议和币种
- 采购入库
- 三类采购对象：机票位、邮轮/游轮舱房、专列铺位
- 机票按航班座位创建库存
- 邮轮按航次舱型创建多条库存
- 专列按班次铺位创建多条库存
- 单价、总成本、系统识别币种
- 采购角色
- 供应商确认状态
- 三期押金/付款阶段
- 节点可调数量和可损规则
- 可用库存、已借调、已售、可释放、已释放
- 单一列表同时展示库存使用和付款情况
- 同航班同日期多批机票库存
- 邮轮舱型库存和价格矩阵
- 专列铺位库存和价格矩阵
- 团期开团资源借调校验
- 列表常规操作不出现手工成交扣减
- 订单成交扣减作为销售中心自动触发的流水结果展示
- 异常库存调整必须记录类型、数量、关联对象和原因
- 库存流水：采购入库、团期借调、订单成交扣减、团期关闭归还、人工调整

不在阶段二验收范围：

- 真实 GDS、船司或铁路系统接口
- 财务付款执行
- 发票、回单、NC 推送
