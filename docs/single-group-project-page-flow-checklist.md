# 单团自组页面执行清单

更新时间：2026-07-08

本文用于承接单团自组后续页面改造，防止上下文压缩后只记得上一个页面，而忘记完整业务目标。

## 1. 总目标

单团自组按“单团项目”组织，不把它做成新的标准线路类型，也不让销售去普通产品预订里手工新建普通订单。

完整操作链路固定为：

```text
创建单团项目
-> 完善需求与客户
-> 维护报价版本
-> 锁定客户确认报价
-> 确认成团
-> 生成项目订单、项目合同草稿、项目团、阶段应收计划
-> 项目团执行名单/资源/成本
-> 项目订单合同签署与收款
-> 回团后成本确认和项目结算
-> 财务承接应收、认款、应付、付款、业务结算、NC
```

一句话边界：

```text
单团详情是主控台；项目团管执行；项目订单管合同收款；财务管认款、应付、付款、结算和 NC。
```

## 4. 单团项目列表验收清单

页面：`merchant/tour/product-custom-list.html`

使用角色：单团销售、单团计调、产品运营。

上游来源：销售线索、客户需求、标准线路详情、历史项目复制。

当前对象：单团项目。

下游承接：单团项目详情、项目订单、项目合同、项目团。

必须可点：

- `引用线路创建独立成团`：打开抽屉，填写客户、线路、方案、日期、人数、差异；若客户已确认锁定报价，生成“待确认成团”的项目行，否则先生成“报价中”的项目行。
- `新建定制需求`：打开抽屉，填写客户需求，生成“需求待完善”的项目行。
- `搜索`：按关键字、类型、来源、负责人和日期过滤当前列表。
- `重置`：清空筛选并恢复全部列表。
- 状态 Tab：按主状态过滤，不改变行操作口径。
- 行内项目名称、`详情`：进入单团项目详情。
- `完善需求`：只出现在“需求待完善”主状态，打开需求处理抽屉；保存后进入报价处理。
- `新建报价`：出现在“报价中”主状态，打开报价抽屉，从需求生成报价版本；客户报价、预算成本和预计毛利必须按交通、酒店、地接、活动、签证保险、服务费等费用项结构化录入，费用包含/不含只作为客户报价单展示口径；保存后仍留在报价中，等待发送或锁定。报价阶段的预算成本只做毛利测算和后续团期成本参考，不直接生成应付或付款。
- `处理报价`、`发送报价`、`锁定报价`、`发起报价审批`：出现在“报价中/报价已锁定”相关主状态，打开报价处理、发送、锁定或审批抽屉；处理报价用于维护已有报价版本，不替代新建报价。
- `确认成团`：只出现在“待确认成团”主状态，弹窗确认后该行变为“已成团”，展示项目订单、项目合同、项目团承接。
- `项目团`、`订单合同`：出现在“已成团”主状态，项目团直接进入项目团详情，订单合同打开项目订单、项目合同、阶段应收承接抽屉。
- `执行处理`、`成本确认`：出现在“执行中”主状态，执行处理聚合名单、资源和出团通知，成本确认进入团期成本承接。
- `回团结算`、`上传验收单`：出现在“已回团”主状态，承接客户验收材料、实际成本复核和结算草稿。
- `查看结算`：出现在“已结算”主状态，反查项目团、项目订单和结算结果。
- `复制`：打开复制抽屉，不只是 toast；复制后生成需求草稿。

列表页不做：

- 不在列表里展开名单明细、资源明细、成本明细、财务流水和结算明细；列表只放高频入口，复杂处理进入抽屉或对应详情页。
- 不跳普通产品预订页新建订单。
- 不在列表里模拟财务认款、付款、NC。

## 5. 单团项目详情验收清单

页面：`merchant/tour/product-custom-detail.html`

使用角色：单团销售、单团计调。

当前对象：单团项目。

详情页必须形成主控台：

- 顶部摘要：主状态、创建路径、客户、负责人、锁定金额、项目团、项目订单、应收、结算承接。
- 主操作：顶部操作栏随主状态变化。报价中只直露“新建报价、锁定报价”；报价已锁定只直露“确认成团”；已成团后只直露“项目团、项目订单”。发送报价、下载报价、报价审批、复制报价、编辑需求、需求确认、项目合同、登记收款、导入名单、新增变更等放入“更多”，并按状态显示。
- Tab：项目总览、需求与客户、报价方案、锁定报价、合同订单、执行资源、成本确认、收款结算、变更记录。
- Tab 操作边界：项目总览、需求、报价、锁定、合同订单、执行资源的标题区不重复顶部主操作；Tab 内只保留行级处理、局部补充和下游反查入口。成本、收款结算、变更属于局部处理域，可保留本 Tab 内的处理按钮。

必须可点：

- 需求：查看需求上下文；编辑需求、生成需求确认单通过顶部“更多”处理。
- 报价：新建报价、复制版本、发送报价、下载报价、AI 报价文案通过顶部操作栏或“更多”处理；报价版本行保留编辑、发送、锁定等版本级操作。
- 锁定：查看锁定快照和承接对象；报价审批、确认成团、新增变更通过顶部操作栏或“更多”处理。
- 确认成团：弹窗展示将生成的项目订单、合同草稿、项目团、阶段应收计划；确认后显示下游入口。
- 合同订单：查看项目合同、查看项目订单、查看应收计划。
- 执行资源：进入项目团、导入名单、维护资源、生成出团通知。
- 成本：查看团期成本、AI 毛利预测、补充成本、确认成本、发起付款申请入口。
- 收款结算：查看应收、登记/认领收款入口、进入对账结算。
- 变更：新增变更、查看变更、提交审批、撤回/作废。

详情页不做：

- 不直接执行财务认款。
- 不直接付款。
- 不直接推送 NC。
- 不把项目团叫“项目团期”或“专属团期”。

## 6. 项目订单详情验收清单

页面：`merchant/sales/orders-detail.html`

菜单位置：销售中心 > 订单管理 > 项目订单详情。

使用角色：单团销售、门市销售主管、订单客服、财务应收协同查看。

当前对象：项目订单，不是普通散拼订单，也不是项目团执行主页面。

上游来源：单团项目确认成团后生成的项目订单、项目合同草稿、项目团和阶段应收计划。

下游承接：销售中心合同管理、销售中心收款认领、履约中心 > 团期管控中的项目团、履约中心 > 团期结算、财务 > 应收管理/收款认款/业务结算。

必须可点：

- 顶部主动作按项目订单主状态显示：待签署显示“发送项目合同”；待首款显示“登记阶段收款”；执行中显示“项目团”；回团待结算显示“查看结算承接”；已完成显示项目团或项目订单资料。
- 顶部“更多”只放当前状态可用的次级动作：项目合同、登记阶段收款、应收调整、导入项目名单、项目团、结算承接、项目变更、项目附件、作废订单。
- 阶段收款 Tab 展示 30% 首款、50% 进度款、20% 尾款等阶段应收计划；不把阶段收款混成普通散拼定金/尾款文案。
- 项目名单 Tab 承接客户企业名单、分组、证件、房型和交通分组；可打开名单维护抽屉。
- 项目合同 Tab 行操作只保留查看和发送/维护，发送进入项目合同发送抽屉，不再堆普通合同的下载、发送、重新生成并列按钮。
- 项目变更/售后 Tab 发起项目变更时，抽屉必须包含变更类型、影响对象、变更后人数、金额影响、客户确认依据和变更说明。
- 发送项目合同抽屉必须核对合同金额、锁定报价、项目团、签署状态，并填写合同模板、签约主体、客户签署人、签署方式、随合同附件和发送备注。
- 登记阶段收款抽屉必须选择收款阶段、收款来源、本次金额、对应应收、凭证/流水和收款说明；提交后承接到收款认领和财务复核，不在订单详情里直接完成财务认款。
- 项目应收调整抽屉必须关联项目变更或客户确认依据，填写调整类型、调整金额、影响阶段和调整原因；审批通过后由财务应收承接。
- 结算承接抽屉必须展示客户验收单、阶段收款、实际成本、项目变更和结算单的缺口及承接页面。

项目订单详情不做：

- 不直接生成项目团执行成本。
- 不直接确认供应商应付、付款或回单。
- 不直接财务认款、开票、对账结算或推送 NC。
- 不作为普通产品预订的新建订单入口。

## 7. 入口承接验收规则

后续每一步执行时，不能只检查页面有没有入口、按钮有没有跳转，还必须检查入口点进去后的内容、字段、默认值、业务逻辑和下游承接是否正确。

统一规则：

- 检查入口是否存在。
- 检查点击后的目标页面、抽屉或弹窗是否进入正确业务对象。
- 检查目标内容是否符合当前来源场景，而不是只复用一个不匹配的通用页面。
- 检查同一业务动作在不同入口的字段和逻辑是否统一。
- 检查目标页里的可点击按钮是否继续符合当前业务路径。

例如“创建独立成团”必须同时验：

- 从 `产品中心 > 自营产品 > 产品管理 > 产品详情` 点击创建独立成团，进入后应生成单团项目并带入产品线路、客户、日期、人数、报价口径。
- 从 `产品中心 > 自营产品 > 单团项目` 点击引用产品线路创建独立成团，抽屉字段和生成逻辑应与产品详情入口一致。
- 两个入口都不能进入普通散拼团期，也不能进入普通产品预订新建订单。
- 创建内容必须要求指定客户；报价锁定和确认成团前生成的是单团项目，确认成团后才承接为项目团，不进入公开渠道售卖。
