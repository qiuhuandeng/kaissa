# 领队管理部系统使用培训稿（场景演示版）

## 一、先讲清领队管理部在系统里的位置

领队管理部不是只维护一个“人员通讯录”，也不是替销售催客户、替产品维护线路、替财务结算费用。领队管理部在整条旅游业务链路中的位置是：管理可派领队资源，确保团期出发前能匹配到合适、合规、有档期的领队，并把领队信息、名单资料和出团通知顺利交给履约执行。

用一句话讲给领队管理部：

领队管理部负责把“可用的人”准确派到“要出发的团”，并保证资质、档期、通知和费用承接有依据。

系统里的领队链路可以按下面这条主线理解：

领队档案 -> 证照协议/培训资料 -> 档期占用 -> 团期出发准备 -> 配领队/更换领队/登记无法出团 -> 名单和资料同步 -> 出团通知 -> 出团中协同 -> 回团资料和费用承接

要先讲清两个层级：

| 层级 | 业务对象 | 领队管理部负责什么 | 下游怎么用 |
| --- | --- | --- | --- |
| 领队资源层 | 领队个人档案 | 姓名、电话、类型、语种、可带区域、证照、协议、结算规则、状态 | 团期配领队时做匹配和校验 |
| 团期执行层 | 某个团期/航次/班期/项目团的领队安排 | 选择领队、集合时间地点、领队备注、无法出团、更换领队 | 出团通知、名单同步、执行协同、费用承接 |

讲师要强调：资源模块管理“这个人能不能派”，团期管控处理“这个团派谁去”。

## 二、培训表达方式

这份材料不按菜单讲，而按领队管理部真实工作场景讲。每个场景统一说明：

- 业务情境：为什么要处理。
- 领队判断：先看资质、档期还是团期状态。
- 系统入口：从领队资源还是团期管控进入。
- 操作步骤：按真实操作顺序演示。
- 状态变化：领队状态和团期领队状态怎么变化。
- 下游承接：名单、通知、履约、成本怎么接。
- 下一步：处理后回到哪里复查。

建议培训顺序：

1. 先讲领队资源档案。
2. 再讲证照、培训、档期、历史带团和费用反查。
3. 再讲团期出发准备时怎么配领队。
4. 再讲邮轮、专列、研学等特殊类型怎么选人。
5. 再讲无法出团和更换领队。
6. 最后讲名单同步、出团通知、回团资料和费用承接。

### 岗位必学清单

| 岗位 | 必须掌握 | 重点理解 | 不作为重点 |
| --- | --- | --- | --- |
| 领队资源管理员 | 新增/编辑领队、维护证照协议、培训资料、可带区域、当前状态、结算规则 | 资源档案决定“能不能被派” | 团期价格、产品线路设计 |
| 领队排班/派团人员 | 从团期管控筛选领队待处理、配领队、更换领队、登记无法出团 | 团期状态、目的地、语言、证照、档期和评分共同决定派谁 | 客户订单收款和合同 |
| 履约/团控配合人 | 查看名单、证件/签证、资源确认、出团通知、标记出发/回团 | 领队安排会影响出团通知和名单同步 | 维护领队长期资质 |
| 业务负责人 | 查看证照预警、不可用领队、关键团期未配领队 | 重点盯临近出发风险 | 逐个录入领队档案 |

### 核心状态流转速查

| 业务对象 | 常见状态流转 | 领队管理部要做什么 |
| --- | --- | --- |
| 领队资源状态 | 空闲 -> 在途 -> 空闲 / 不可用 | 维护当前可派状态和不可用原因 |
| 证照协议 | 有效 -> 到期预警 -> 不可用 | 到期前续证，避免团期无法派 |
| 团期主状态 | 销售中 -> 出发准备 -> 出团中 -> 已回团 -> 已结束 | 出发准备和出团中是领队高频处理阶段 |
| 团期领队状态 | 未安排 -> 已配领队 -> 无法出团 -> 重新配领队 | 团期临近时必须处理未安排和无法出团 |
| 出团通知 | 未生成 -> 待发送 -> 已发送/部分确认 | 领队信息会进入通知内容 |
| 费用承接 | 未到节点 -> 费用待确认 -> 成本/补贴承接 | 回团后领队费用进入成本或工资补贴承接 |

## 三、场景一：新领队入库，建立可派档案

### 业务情境

公司新签了一名外聘领队，可以带欧洲线和研学线，会英语和德语，需要录入系统并设置结算规则。

### 领队管理员先判断什么

录档案前先确认：

- 是全职还是外聘。
- 手机号和紧急联系人是否准确。
- 可带区域和语种是否清楚。
- 导游证、签证、保险、外聘协议是否有效。
- 是否有专项培训，如邮轮、研学、MICE。
- 结算规则是什么：工资补贴承接还是按天结算。
- 当前状态是否可以派团。

### 系统入口

从领队资源列表点击“新增领队”，或选择已有领队点击编辑。

### 操作步骤

第一步，填写基础档案。

包括姓名、手机号、语种、可带区域、领队类型、导游证号、导游证有效期、外聘协议有效期、结算规则、紧急联系人和当前状态。

第二步，维护当前状态。

常见状态：

- 空闲：当前可以进入派团候选。
- 在途：当前已在团期中，不应重复派。
- 不可用：证照、档期或个人原因导致暂不可派。

第三步，保存档案。

保存后可以在领队资源列表按姓名、手机号、目的地资质、语种和当前状态筛选。

### 状态如何变化

未入库 -> 领队档案 -> 空闲/不可用 -> 可进入团期配领队候选

### 下游承接

团期配领队时，系统会根据目的地、语言、档期、证照和评分排序推荐。档案不完整，会影响团期可派判断。

### 常见错误

- 只录姓名电话，不录可带区域和语种。
- 外聘领队不录协议有效期。
- 导游证快到期仍标记空闲。
- 结算规则不清，回团后费用无法承接。

## 四、场景二：每天检查可派领队和证照预警

### 业务情境

暑期团期密集，领队管理部每天要先看哪些人可派、哪些人证照到期、哪些人当前在途。

### 领队管理员先判断什么

每天先看：

- 空闲领队数量。
- 不可用领队原因。
- 导游证、签证、保险、协议是否到期预警。
- 当前在途领队回团日期。
- 高评分领队是否有档期冲突。
- 特殊资质是否满足：邮轮、研学、专列、MICE。

### 系统操作步骤

第一步，进入领队资源列表。

使用筛选：

- 姓名/手机号。
- 目的地资质。
- 语种。
- 当前状态。

第二步，查看列表核心字段。

列表重点看：

- 领队姓名和编号。
- 类型。
- 联系方式。
- 目的地/语言。
- 证照协议。
- 结算规则。
- 当前状态。
- 最近带团。
- 评分。

第三步，打开领队详情。

详情页签用于更完整判断：

- 基础信息：身份、电话、类型、语种、可带区域、结算规则。
- 证照协议：导游证、签证、保险、合作协议及其影响。
- 培训资料：欧洲、邮轮、MICE 等培训是否有效。
- 档期占用：已配团期和派团状态。
- 历史带团：回团日期、目的地、游客数、评分。
- 费用反查：领队费用、成本状态和财务承接。

### 状态如何变化

证照有效 -> 可派  
证照到期预警 -> 不可用/待续证 -> 续证后可派  
空闲 -> 已配团期 -> 在途 -> 回团后空闲

### 下游承接

证照和培训资料直接影响团期配领队候选。如果领队有到期预警或档期冲突，系统在配领队时会提示或禁用。

## 五、场景三：团期进入出发准备，安排领队

### 业务情境

某个欧洲团期已经销售中并临近出发，名单、资源和通知都在准备，但领队还未安排。团期管控出现“领队未安排”缺口。

### 领队排班先判断什么

派团前先看：

- 团期主状态是否已到出发准备。
- 产品类型：普通团期、邮轮航次、专列班期、研学营期、项目团。
- 目的地和线路。
- 出发/回团日期。
- 已售人数。
- 领队要求：语言、区域、经验、特殊资质。
- 候选领队是否空闲、证照有效、档期不冲突。
- 集合时间和集合地点是否确定。

### 系统入口

从团期管控进入，而不是从资源档案里“给人找团”。常见入口：

- 团期列表筛选关键缺口：领队待处理。
- 出发准备状态的团期列表操作：配领队。
- 团期详情的执行准备区域。

### 操作步骤

第一步，筛选领队待处理团期。

在团期管控中按关键缺口或领队状态筛选，重点看出发准备和出团中阶段。

第二步，打开配领队抽屉。

抽屉会显示当前团期摘要：

- 团期编号。
- 产品。
- 使用线路。
- 出发日期。
- 已售人数。

第三步，看系统推荐规则。

普通团期按目的地、语言、档期、证照和评分排序。邮轮航次按邮轮经验、档期、证照和港口协调能力排序。专列班期按专列经验、档期、证照和站点协调能力排序。

第四步，选择领队。

候选卡片会显示：

- 领队姓名和电话。
- 全职或外聘。
- 可带区域。
- 语种。
- 证照有效期。
- 评分。
- 推荐、证件预警、档期冲突或目的地不匹配。

第五步，填写集合时间、集合地点和领队备注。

这些信息会影响出团通知和领队执行准备。

第六步，保存配领队。

保存后团期领队状态变为已配领队。

### 状态如何变化

团期出发准备 + 领队未安排 -> 配领队 -> 已配领队

### 下游承接

- 出团通知读取领队姓名、电话、集合时间和地点。
- 名单同步给领队时，领队可以看到游客、证件和特殊需求。
- 回团后领队费用进入团期成本或工资补贴承接。

## 六、场景四：邮轮、专列、研学和项目团的领队安排差异

### 产品类型差异速查

| 团期类型 | 领队角色叫法 | 选人重点 | 出团通知重点 |
| --- | --- | --- | --- |
| 普通团期 | 领队 | 目的地、语言、导游证、档期、评分 | 机场集合、护照签证、领队联系方式 |
| 邮轮航次 | 邮轮管家/领队 | 邮轮经验、登离船港口、船方培训、港口协调 | 登船港、登船时间、船票、岸上游确认 |
| 专列班期 | 随车领队 | 专列经验、始发终到、上下车站、铺位协调 | 发车站、车厢铺位、实名制证件 |
| 研学营期 | 带队老师/领队 | 研学经验、安全培训、学生管理、监护材料 | 集合点、监护授权、学生名单和特殊需求 |
| 项目团 | 项目领队/随团工作人员 | 客户接待标准、活动经验、语言和现场协调 | 客户联系人、活动流程、项目特殊要求 |

### 讲法

不要只说“点击配领队”。要让学员理解：不同团型对领队的能力要求不同。系统里的候选推荐是为了降低派错人的风险，但最终还是要由业务人员按团型和客户要求判断。

### 邮轮演示重点

邮轮航次要看：

- 船方培训是否有效。
- 登船港和离船港。
- 岸上游安排。
- 船票和登船资料。
- 港口集合和登船节奏。

### 专列演示重点

专列班期要看：

- 始发站和终到站。
- 上下车站。
- 车厢和铺位。
- 实名制证件。
- 随车服务和站点协调。

### 研学演示重点

研学营期要看：

- 学生名单和监护人信息。
- 监护授权。
- 安全培训。
- 特殊需求和医疗备注。
- 带队老师和领队职责边界。

## 七、场景五：领队无法出团，需要登记并重新配领队

### 业务情境

团期已经配了领队，但领队临时因证照、身体、档期或突发原因无法出团。系统不能只口头通知，需要登记原因并重新安排。

### 领队排班先判断什么

- 当前团期是否已进入出发准备或出团中。
- 原领队是否已经进入出团通知。
- 名单是否已经同步给原领队。
- 是否需要补发通知给游客。
- 新领队是否满足目的地、语言、资质和档期。
- 费用承接是否会发生变化。

### 系统操作步骤

第一步，在团期列表或详情打开领队处理。

已配领队的团期在更多操作中可以选择“更换领队”或“登记无法出团”。

第二步，登记无法出团。

填写无法出团原因。系统把团期领队状态更新为无法出团。

第三步，重新打开配领队。

选择新的可派领队，重新填写集合时间、集合地点和领队备注。

第四步，核对出团通知。

如果通知已发送，需要补发或更新通知，确保游客和销售看到新的领队信息。

第五步，核对名单同步。

如果名单已经同步给原领队，需要重新同步给新领队。

### 状态如何变化

已配领队 -> 登记无法出团 -> 无法出团 -> 重新配领队 -> 已配领队

### 下游承接

更换领队会影响出团通知、名单同步、现场执行和费用承接。不能只在备注里写一句“换人了”。

## 八、场景六：出团前同步名单和生成出团通知

### 业务情境

团期已配领队，出发前需要让领队掌握游客名单、证件缺口、特殊需求和集合通知。

### 领队管理部先判断什么

- 游客名单是否已导入。
- 证件/签证是否完整。
- 是否有老人、儿童、素食、行动不便等特殊需求。
- 交通、酒店、船票、铺位等资源是否确认。
- 集合时间和地点是否准确。
- 通知模板是否符合团型。

### 系统入口

从团期管控的出发准备进入：

- 名单。
- 证件/签证。
- 资源/票务确认。
- 出团通知。
- 配领队。

### 操作步骤

第一步，看名单状态。

名单管理里按团期分组展示游客。普通团期看姓名、证件、手机、房型、特殊需求和资料状态。邮轮看舱型/舱号和登船资料。专列看席别/铺位和上下车站。研学看监护人手机、住宿、特殊需求和安排信息。

第二步，做名单核验。

可使用 AI 名单核验发现缺证件、证件格式异常、重复游客、手机号缺失和特殊需求风险。

第三步，同步名单给领队。

同步给领队后，团期名单状态更新。领队获得游客姓名、订单、电话、资料状态和特殊需求。

第四步，生成或发送出团通知。

出团通知会读取团号、产品名称、开始日期、集合/登船/发车信息、负责人、发送人数和确认人数。

第五步，核对通知内容。

通知内容要包含：

- 产品名称。
- 出发日期。
- 集合地点。
- 集合时间。
- 领队姓名和电话。
- 证件和关键提醒。
- 发送渠道。

### 状态如何变化

名单待导入 -> 名单已导入/资料待补 -> 名单同步给领队  
通知未生成 -> 待发送 -> 已发送/部分确认

### 下游承接

游客收到通知，销售能跟进客户确认，领队能提前掌握名单和特殊需求，履约能继续处理资源和出团执行。

## 九、场景七：出团中和回团后的领队承接

### 业务情境

团期已出发，领队在途。回团后需要把出团资料、服务情况和领队费用交给后续成本结算。

### 领队管理部先判断什么

出团中看：

- 领队是否仍在途。
- 是否发生更换领队。
- 是否有游客名单或资源变更。
- 是否有紧急事项需要记录到团期执行。

回团后看：

- 是否已标记回团。
- 出团资料是否完整。
- 领队费用是否需要确认。
- 全职领队走工资补贴还是外聘领队按天结算。
- 是否有客户评价或领队评分。

### 系统操作步骤

第一步，在团期管控查看出团中团期。

出团中阶段可进入执行详情，必要时更换领队。一般不再做普通配领队，除非出现异常。

第二步，回团后查看出团资料。

已回团团期在更多操作中可以查看出团资料，成本相关进入团期成本。

第三步，在领队详情看历史带团。

历史带团展示回团日期、团期、目的地、游客数和评分，用于后续派团参考。

第四步，费用反查。

费用反查展示费用项目、关联团期、确认金额、成本状态和财务承接。全职可能是工资补贴承接，外聘可能按天结算。

### 状态如何变化

已配领队 -> 出团中/在途 -> 已回团 -> 费用待确认/工资补贴承接 -> 历史带团沉淀

### 下游承接

回团后的费用和成本确认进入团期成本和财务模块。领队资源详情只做反查和依据展示，不在这里完成财务付款。

## 十、场景八：领导每天怎么查领队风险

### 业务情境

领队管理负责人每天早上要快速判断近期团期是否有派团风险。

### 每天优先看什么

| 检查项 | 系统看哪里 | 判断标准 |
| --- | --- | --- |
| 临近出发未配领队 | 团期管控按关键缺口筛选领队待处理 | 出发准备阶段有未安排或无法出团 |
| 证照到期预警 | 领队资源列表和详情证照协议 | 导游证、签证、保险、协议临期 |
| 档期冲突 | 领队详情档期占用 | 同一时间不能重复派 |
| 特殊团型缺专长 | 团期类型和候选推荐 | 邮轮、专列、研学、MICE 需对应经验 |
| 通知未发送 | 出团通知列表 | 待发送、部分确认、未生成 |
| 名单资料缺失 | 名单管理和团期关键缺口 | 缺证件、实名制资料缺、登船资料缺 |
| 费用未承接 | 领队详情费用反查/团期成本 | 回团后领队费用未确认 |

### 处理顺序

优先级建议：

1. 明天到 7 天内出发且未配领队。
2. 已配领队但证照预警或无法出团。
3. 通知已发送但更换过领队。
4. 邮轮、专列、研学等特殊团型缺专长领队。
5. 回团后费用未确认。

## 十一、现场演示台本

### 第一轮：建立领队资源和团期安排的关系

先打开领队资源列表，再打开团期管控。

讲师话术：

领队资源列表回答“这个人能不能派”；团期管控回答“这个团派谁去”。两个入口不能混在一起讲。

### 第二轮：新增领队档案

演示新增领队：

1. 填姓名、电话、语种、可带区域。
2. 选择全职或外聘。
3. 填导游证和协议有效期。
4. 填结算规则。
5. 设置当前状态。
6. 保存。

### 第三轮：查看领队详情

打开张建国领队详情，依次讲：

1. 基础信息。
2. 证照协议。
3. 培训资料。
4. 档期占用。
5. 历史带团。
6. 费用反查。

### 第四轮：团期出发准备配领队

进入团期管控，筛选领队待处理。

演示：

1. 打开配领队。
2. 看团期摘要。
3. 看推荐规则。
4. 选择推荐领队。
5. 填集合时间地点和备注。
6. 保存配领队。

### 第五轮：邮轮和专列配领队差异

分别打开邮轮航次和专列班期。

讲师强调：

邮轮看船方培训、港口协调和登船资料；专列看铺位、上下车站和随车经验。

### 第六轮：登记无法出团并重新配领队

选择已配领队团期，演示：

1. 登记无法出团。
2. 填写原因。
3. 团期变为无法出团。
4. 重新配领队。
5. 检查是否需要补发通知和重新同步名单。

### 第七轮：名单同步和出团通知

进入名单管理和出团通知：

1. 查看按团期分组名单。
2. 讲证件缺失、实名制资料、舱号/铺位、特殊需求。
3. 同步名单给领队。
4. 生成出团通知。
5. 核对领队、集合时间地点和关键提醒。

### 第八轮：回团后费用和历史沉淀

进入已回团团期和领队详情费用反查：

1. 查看出团资料。
2. 进入团期成本。
3. 回到领队详情看历史带团。
4. 查看领队费用承接状态。

## 十二、常见错误提醒

| 错误做法 | 为什么错 | 正确做法 |
| --- | --- | --- |
| 只在领队档案里写备注“可派欧洲” | 配领队时无法按字段筛选 | 维护可带区域、语种、证照和培训资料 |
| 导游证到期仍保持空闲 | 团期会误派不合规领队 | 标记不可用或处理续证 |
| 从资源档案里直接决定某个团派谁 | 离开团期上下文，容易忽略日期和团型 | 从团期管控进入配领队 |
| 只口头通知更换领队 | 出团通知和名单同步仍是旧人 | 登记无法出团，再重新配领队 |
| 更换领队后不补发通知 | 游客和销售看到旧联系方式 | 更新或补发出团通知 |
| 名单未齐就同步给领队 | 领队拿到的信息不完整 | 先处理证件、实名制、舱号/铺位等缺口 |
| 把领队费用在资源详情里结算 | 财务承接不清 | 资源详情只反查，费用进入团期成本/财务 |
| 特殊团型按普通团选人 | 邮轮、专列、研学风险高 | 按团型经验和专项培训匹配 |

## 十三、培训收口

领队管理部培训结束时，学员要能回答五个问题：

- 这个领队现在能不能派，依据是什么？
- 这个团期是否已经进入需要配领队的阶段？
- 当前团型对领队有什么特殊要求？
- 配领队后，名单和出团通知是否已经承接？
- 回团后，领队费用和历史带团记录在哪里反查？

最后一句话：

领队管理的关键不是“把名字填上去”，而是让每个即将出发的团，都能匹配到合适、合规、有档期、信息同步到位的领队。
