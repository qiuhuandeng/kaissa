# CSS 区域索引

本文件用于帮助 AI 和人工维护者快速定位 `shared/style.css`。当前项目仍保留单一运行文件，本文档只做索引，不改变任何页面引用。

当前体量：

- 文件：`shared/style.css`
- 行数：约 68,860 行
- 角色：现阶段仍作为所有 HTML 页面的演示运行样式

## 使用原则

后续修改页面时按以下顺序查找：

1. 先读 `docs/AI_PAGE_RULES.md`。
2. 需要复制页面结构时读 `docs/HTML_PATTERNS.md`。
3. 需要找现有样式时读本文件。
4. 优先使用公共规范区的 class。
5. 只有公共规范无法覆盖时，才进入业务前缀区补页面样式。

不要把新样式随手追加到 `shared/style.css` 末尾。新增页面级样式必须有页面前缀。

## 修改建议

### 新增列表页

优先参考：

- `docs/HTML_PATTERNS.md` 的“后台业务列表页”
- `shared/style.css:43686-45340`

不要优先复制旧页面的 `nth-child` 列宽。

### 新增抽屉

优先参考：

- `docs/HTML_PATTERNS.md` 的“业务处理/状态推进抽屉”或“详情/速览/记录抽屉”
- `shared/style.css:48513-48966`

不要在标题区写业务说明，不要新增抽屉解释性文案。

### 修改旧页面

如果只是修局部展示问题：

- 先在旧页面前缀范围内最小修改。
- 不把旧局部规则升级为全局规则。

如果是页面整体优化：

- 先按 `docs/AI_PAGE_RULES.md` 重判角色、对象、入口和下游。
- 列表迁移到 `.list-surface`。
- 抽屉迁移到 `.drawer-object-summary` + `.drawer-section`。
