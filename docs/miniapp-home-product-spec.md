# 凯撒优选小程序首页产品稿

## 1. 页面定位

页面路径：`/pages/home/index`

首页处在 C 端购买旅程的第一步，目标是把用户从种草带到可购买的产品集合或具体产品详情。

上游来源：

- 小程序自然打开
- 微信分享卡片
- 活动 Banner 链接
- 门店专属前台入口
- AI助手推荐后回到首页

下游去向：

- 产品列表：搜索词、目的地、主题、出境热门、探索 Tab
- 产品详情：Banner 或好看世界线路卡片
- AI助手：Banner、底部 Tab、搜索无结果时引导
- 我的旅程：底部 Tab
- 我的：底部 Tab

主要用户：C 端消费者，包括有明确目的地的人、被内容种草的人、需要客服/AI辅助决策的人、已下单回访的人。

最高频任务：

1. 搜索目的地或线路
2. 从热门目的地/主题快速进入列表
3. 浏览平台推荐线路并进入详情
4. 进入我的旅程查看已购订单

首页不处理下单、付款、退款、合同、发票等深业务动作，只提供购买入口和服务入口。

## 2. 首页设计原则

- 转化率优先：搜索、目的地、热门线路必须比装饰内容更容易点到
- 操作路径最短：首页所有核心入口最多一次点击进入产品列表或详情
- 视觉高级但克制：大图建立情绪，卡片承接信息，不堆满营销贴纸
- 真实旅游决策优先：用户先按目的地、主题、出发时机和价格建立兴趣
- AI服务可见但不喧宾夺主：AI作为辅助咨询入口，不替代主购买链路
- 多主体可替换：Banner、皮肤色、运营活动位预留门店/主体差异

## 3. 首屏结构定稿

首屏推荐结构：

```text
状态栏 + 小程序胶囊
品牌标题/城市入口
多主体皮肤 Banner
智能搜索入口（Banner 下方，滚动后吸顶）
目的地快速入口（首屏露出完整或大半）
运营活动卡位（会员/限时活动，露出顶部）
```

关键调整：

- 顶部可以保留城市和搜索图标，但真正的主搜索入口放在 Banner 下方。
- Banner 负责情绪和活动，不承担所有导航任务。
- 搜索入口要比分类卡更靠前，因为“有明确意图的用户”转化更高。
- 目的地入口首屏必须露出，不能被活动 Banner 挤到过深位置。
- 底部 Tab 固定 5 项，不能继续用 3 项结构。

## 4. 模块顺序

| 顺序 | 模块 | 作用 | 点击去向 | P0 决策 |
| --- | --- | --- | --- | --- |
| 1 | 多主体皮肤 Banner | 建立品牌感和活动主推 | 产品详情、活动落地页、AI助手 | 做轮播视觉，先不做后台配置 |
| 2 | 智能搜索入口 | 承接明确需求用户 | 产品列表 | 做搜索框、热门词、联想样式 |
| 3 | 目的地快速入口 | 快速按区域/主题找产品 | 产品列表 | 10 个入口，支持目的地和主题混排 |
| 4 | 运营活动卡位 | 承接会员积分/限时优惠 | 活动页或产品列表 | P0 做一个横幅卡，不深做会员 |
| 5 | 精选旅游 | 按主题种草 | 产品列表 | 横向主题卡，突出产品数量和季节性 |
| 6 | 出境热门 | 按国家/目的地聚合 | 产品列表 | 展示最近出发日和起步价 |
| 7 | 好看世界 | 热门线路商品流 | 产品详情 | 大图卡片纵向流，每屏露出 1.5 张 |
| 8 | 底部 Tab | 全局导航 | 首页/探索/旅程/AI/我的 | 固定 5 项 |

## 5. 模块详细规格

### 5.1 顶部导航

展示内容：

- 城市入口：默认北京
- 品牌标题：凯撒优选
- 小程序胶囊：更多/关闭视觉模拟

交互：

- 点击城市：P0 可打开城市选择抽屉或跳转占位
- 点击顶部搜索图标：聚焦主搜索入口

P0 边界：

- 不做真实定位授权
- 不做多城市价格真实切换

### 5.2 多主体皮肤 Banner

展示内容：

- 大图或视频封面
- 活动主标题
- 活动副标题
- 可选角标：门店专属、暑期、邮轮季、会员专享

Banner 示例：

- 心向蔚蓝 活力海洋
- 暑期亲子研学季
- 欧洲深度游早鸟计划
- 理想号邮轮假期

交互：

- 点击 Banner 进入产品详情、产品列表、活动页或 AI助手

P0 边界：

- 先用静态轮播数据
- 不做加盟门店后台皮肤配置联动

### 5.3 智能搜索入口

展示位置：Banner 下方，滚动后吸顶。

展示内容：

- 输入框占位：想去哪里？试试“北欧极光”“亲子研学”
- 语音按钮
- 热门搜索词：欧洲、邮轮、亲子、春节、冰岛、日本

联想内容：

- 线路名：北欧四国+冰岛13天
- 目的地：意大利、埃及、澳大利亚
- 主题词：亲子研学、银发康养、邮轮度假

交互：

- 输入关键词后进入搜索结果态
- 点击热门词进入产品列表，并携带筛选条件
- 无结果时展示 AI助手入口：“让AI帮你找合适线路”

P0 边界：

- 做前端联想样式和示例数据
- 不接真实搜索服务
- 不做语音识别真实能力

### 5.4 目的地快速入口

展示形式：2 行 5 列图标入口。

P0 入口建议：

- 欧洲
- 日本
- 东南亚
- 中东非
- 大洋洲
- 美洲
- 国内游
- 亲子研学
- 环球邮轮
- 全部旅游

交互：

- 点击进入产品列表
- 自动带入筛选条件

筛选参数示例：

```text
欧洲 -> destinationRegion=europe
亲子研学 -> theme=family-study
环球邮轮 -> productType=cruise
全部旅游 -> no preset filter
```

P0 边界：

- 不做展开完整洲际分组
- 不做复杂目的地树

### 5.5 运营活动卡位

作用：承接会员积分、限时优惠、门店专属活动。

展示内容：

- 活动图
- 主标题
- 副标题
- 行动按钮

示例：

- 会员积分专享，同款更优惠
- 暑期亲子报名礼
- 邮轮阳台房限量升级

交互：

- 点击进入活动页或产品列表

P0 边界：

- 活动页可以先不做，只跳产品列表或占位页
- 不做真实积分兑换流程

### 5.6 精选旅游

作用：按出行主题种草，适合无明确目的地的用户。

展示形式：横向滑动主题卡片。

卡片字段：

- 主题封面大图
- 主题名称
- 产品数量
- 一句话主题利益点

主题建议：

- 亲子研学
- 蜜月浪漫
- 银发康养
- 深度文化
- 自然探索
- 邮轮度假
- 极地冰雪
- 美食之旅

交互：

- 点击进入产品列表，自动带入主题筛选条件

P0 边界：

- 做 6 到 8 张主题卡即可
- 不做季节算法，只用运营配置表达

### 5.7 出境热门

作用：按国家/目的地聚合，承接明确目的地但不确定线路的用户。

展示形式：横向目的地卡或 2 列轻卡。

卡片字段：

- 国家/目的地名称
- 代表性景点图
- 热门城市标签
- 最近可出发日
- 最低起步价

示例：

- 意大利：罗马/佛罗伦萨/威尼斯，4月11日可出发，¥28999起
- 北欧：冰岛/挪威/芬兰，5月3日可出发，¥51999起
- 埃及：开罗/卢克索/红海，4月26日可出发，¥16999起

交互：

- 点击进入产品列表，并过滤目的地
- “查看更多国家”进入完整目的地列表或产品列表

P0 边界：

- 排序先按模拟热度
- 不做真实销量动态排序

### 5.8 好看世界

作用：用热销/口碑线路推动用户进入详情。

展示形式：纵向大图卡片，每屏露出约 1.5 张，鼓励下滑。

卡片字段：

- 高质量横版封面图
- 收藏按钮
- 方案类型标签：3种配置可选、舒适/豪华
- 线路名称
- 行程天数
- AI生成卖点一句话
- 起步价
- 最近出发日
- 已售人数
- 余位紧张提示

卡片示例：

```text
北欧四国+冰岛13天11晚
自然巨幕 · 峡湾巡游 · 极光季精选
经济方案起 ¥51999/人
最近出发 5月3日 · 已售 428 人 · 剩余 4 席
```

交互：

- 点击卡片进入产品详情
- 点击收藏只改变收藏态，P0 不做真实账号收藏落库

P0 边界：

- 下拉刷新可先不做真实换一批
- AI卖点使用静态文案模拟

### 5.9 底部 Tab

固定 5 项：

| Tab | 路径 | 作用 |
| --- | --- | --- |
| 首页 | `/pages/home/index` | 种草和入口 |
| 探索 | `/pages/explore/list` | 搜索筛选产品 |
| 我的旅程 | `/pages/trips/list` | 订单服务 |
| AI助手 | `/pages/ai/chat` | 导购和出行服务 |
| 我的 | `/pages/profile/index` | 个人中心 |

P0 要求：

- 首页高亮
- 我的旅程可显示未付款/待办角标
- AI助手使用有识别度但不浮夸的图标

## 6. 页面跳转规则

| 入口 | 目标 | 携带参数 |
| --- | --- | --- |
| 搜索关键词 | 产品列表 | `keyword` |
| 热门搜索词 | 产品列表 | `keyword` 或 `theme` |
| 目的地入口 | 产品列表 | `destinationRegion` / `destinationCountry` |
| 主题入口 | 产品列表 | `theme` |
| 出境热门国家 | 产品列表 | `destinationCountry` |
| 好看世界线路卡 | 产品详情 | `routeId` |
| Banner 产品活动 | 产品详情或产品列表 | `routeId` / `campaignId` |
| AI助手入口 | AI助手 | `scene=guide` |
| 我的旅程 Tab | 我的旅程列表 | 无 |

## 8. P0 验收标准

首页视觉：

- 首屏能清楚看到品牌、Banner、搜索和目的地入口
- 页面不是纯营销海报，能明显承接购买路径
- 底部 Tab 是 5 项，且命名与 P0 总图一致

首页转化：

- 任一核心入口都能说明跳到列表还是详情
- 搜索入口比运营活动更靠前
- 好看世界卡片能明确展示价格、最近出发日和方案类型

首页边界：

- 不做真实定位、真实搜索、真实推荐算法
- 不做会员积分兑换闭环
- 不做后台多主体配置联动
- 不做订单、付款、退款、发票动作

## 9. 下一步进入代码前的改造清单

基于当前首页视觉原型，下一次代码改造应完成：

1. 底部 Tab 从 3 项改为 5 项：首页、探索、我的旅程、AI助手、我的
2. 将主搜索入口从顶部胶囊弱入口升级为 Banner 下方搜索条
3. 调整首页模块顺序为：Banner、搜索、目的地、活动卡、精选旅游、出境热门、好看世界
4. 将“精选旅游”从商品瀑布改为主题横滑卡
5. 将“好看世界”作为线路商品流，展示方案类型、最近出发日、已售和价格
6. 所有入口补齐目标路径和参数语义
7. 保留当前高级海洋度假视觉，但减少底部遮挡和首屏拥挤
