# 财务部门系统培训方案（业财一体化主链版）

## 一、财务部门在系统里的定位

财务部门这场培训不能讲成“财务模块功能介绍”，而要讲成：

**这套 ERP 怎么把业务数据自动变成财务可核对、可结算、可入账、可推 NC 的链路。**

财务不是单纯使用者，而是整个系统链路的最终验收者。

销售、产品、计调、履约做完各自业务后，系统最终要回答财务关心的问题：

- 这笔钱该不该收；
- 钱有没有到账；
- 到账后认到哪张订单、哪张应收；
- 这笔成本有没有业务依据；
- 该不该给供应商付款；
- 发票、回单、汇率、结算、NC 是否能对上；
- 报表里的数能不能穿透到源单。

所以财务培训的核心不是教财务“在哪里点按钮”，而是让财务理解：

**业务部门负责把业务事实录准，系统负责把业务事实自动沉淀成财务对象，财务负责核对、确认、处理异常、完成入账和对外支付。**

### 财务培训要讲清七件事

| 关键问题            | 培训要讲清什么                                 |
| --------------- | --------------------------------------- |
| 哪些业务动作会自动形成财务数据 | 订单、收款、退款、成本确认、结算申请、付款申请等动作如何触发财务对象      |
| 财务数据从哪里来        | 订单、合同、团期、项目、服务单、到账流水、供应商账单、发票、回单        |
| 财务需要核什么         | 金额、对象、主体、币种、汇率、附件、票据、账户、NC 口径           |
| 哪些能自动处理         | 自动生成应收、自动识别流水、自动认款、自动生成开票任务、自动生成 NC 任务  |
| 哪些异常必须财务介入      | 认款不匹配、成本依据缺失、供应商账单差异、账户异常、发票不一致、NC 推送失败 |
| 上游数据不准怎么办       | 财务通过退回、卡控、异常提示倒逼业务补齐源头数据                |
| 最终如何闭环          | 结算、付款、发票、回单、NC 凭证和报表如何形成完整链路            |

### 培训语气

这场培训要反复强调：

**系统不是让财务多录一遍业务数据，而是让业务数据在源头结构化，财务从人工录入变成核对、例外处理和规则管理。**

财务不应该替业务补订单、补名单、补成本依据、补供应商账单。财务看到的是业务链路沉淀下来的财务对象，财务要做的是核对、确认、退回、异常处理和最终入账。

## 二、业财一体化总链路

财务要先从全链路看系统，而不是从财务菜单看系统。

系统里的业财一体化主链可以这样讲：

```text
产品/线路/团期/项目/服务单
-> 销售订单/合同/售后
-> 应收/到账流水/认款/预收/退款
-> 履约成本/供应商账单/应付
-> 付款申请/审批/付款执行/回单
-> 业务结算/门店结算/渠道结算
-> 发票/NC/报表/关账
```

这条链路里，业务部门录入的是业务事实，系统沉淀的是财务对象，财务最终验收的是资金、票据、结算、凭证和报表。

### 和业务部门培训的区别

| 业务培训               | 财务培训                         |
| ------------------ | ---------------------------- |
| 销售讲客户怎么报价、下单、收款、售后 | 财务讲订单如何生成应收，收款如何认款，退款如何冲减    |
| 产品讲产品怎么上架、价格怎么维护   | 财务讲价格、退改、渠道政策如何影响应收、退款、发票和结算 |
| 计调讲团期怎么执行、成本怎么形成   | 财务讲成本如何进入应付，付款如何审批和执行        |
| 领队/履约讲出团和回团资料      | 财务讲回团资料、供应商账单和成本依据如何进入结算     |

财务部门听这场培训时，要把每个业务动作翻译成财务对象：

- 订单确认：生成应收；
- 客户付款：进入到账流水；
- 认款确认：核销应收；
- 售后退款：形成退款单和应收冲减；
- 团期回团：成本进入复核；
- 供应商账单确认：形成应付；
- 付款申请审批通过：进入付款执行；
- 付款完成：回单核销；
- 发票处理完成：进入税务和 NC；
- 结算确认：进入报表和 NC。

## 三、财务核心对象

财务培训前必须先统一对象语言。否则财务很容易把“到账流水、认款、收款凭证、应收、结算”混在一起。

| 财务对象  | 是什么                        | 从哪里来                              | 到哪里去                 |
| ----- | -------------------------- | --------------------------------- | -------------------- |
| 应收    | 客户、门店、渠道或项目应该付给公司的金额       | 订单、合同、项目阶段款、补差、应收调整               | 收款认款、发票、结算、NC        |
| 到账流水  | 银行、银联、POS、支付平台或上传水单形成的资金流水 | 银行接口、银联水单、POS、微信支付宝、门店上传、Excel 导入 | 流水识别、收款认款、预收、预存、退款退回 |
| 认款    | 确认一笔或多笔流水核销哪张应收            | 到账流水、销售收款认领、系统自动匹配                | 收款凭证、订单收款状态、NC       |
| 预收    | 客户或订单层面的余额                 | 超收、企业预付款、退款转预收、转款剩余               | 新订单抵扣、退款、NC          |
| 预存    | 门店、渠道或客户账户余额               | 门店充值、渠道预存、账户扣款、退款退回               | 订单扣款、门店对账、资金池、NC     |
| 退款    | 已审批的退款执行对象                 | 售后退款、应收冲减、预存退款                    | 退款执行、银行回单、发票红冲、NC    |
| 供应商账单 | 供应商提供的成本账单                 | 供应商上传、邮件导入、接口、履约成本                | 应付确认、进项发票、结算         |
| 应付    | 已确认的付款义务                   | 成本确认、供应商账单、采购批次、服务单代垫             | 付款申请、付款执行、发票、结算、NC   |
| 付款申请  | 基于应付或预付节点发起的付款申请           | 业务侧成本页、应付单、合同节点、供应商账单             | 审批、付款执行              |
| 付款    | 出纳实际付款结果                   | 已审批付款申请                           | 银行流水、回单、应付状态、NC      |
| 回单    | 付款或退款后的银行结果证明              | 银行回单、银企直联、人工上传                    | 回单核销、付款完成、NC         |
| 结算    | 对团期、项目、订单、门店、渠道的财务确认       | 收入、成本、退款、应付、付款、发票、回单              | 报表、NC、关账             |
| 发票    | 销项开票、进项收票、红冲、验真等税务对象       | 应收、收款、应付、付款、退款、结算                 | 税务台账、NC、结算风险         |
| NC    | 财务凭证推送任务和结果                | 收款、退款、应付、付款、结算、发票、调整              | NC 返回号、凭证状态、失败重推     |

财务培训时要特别强调三个区别：

1. 到账流水不是收款凭证，认款确认后才形成收款凭证。
2. 付款申请不是付款结果，出纳执行并回单核销后才算付款完成。
3. 结算确认不是业务执行，结算是财务对已发生业务数据的复核和锁定。

## 四、财务培训按三条主链讲

财务部门真正关心的 ERP 主干是三条：

### 第一条：收入链路

```text
订单 -> 合同/应收 -> 到账流水 -> 收款认款 -> 应收核销 -> 发票 -> 收入确认/NC
```

收入链路要讲清：

- 销售订单确认后，系统如何自动形成应收；
- 客户付款到账后，如何进入到账流水；
- 银行、银联、POS、门店上传水单如何被系统识别；
- 认款后如何核销订单应收；
- 超收、短收、拆分收款、合并收款如何处理；
- 发票如何根据应收、收款和客户抬头自动生成任务；
- 收入确认和 NC 凭证如何承接。

### 第二条：成本付款链路

```text
团期/项目/服务成本 -> 应付确认 -> 付款申请 -> 审批 -> 付款执行 -> 回单 -> 发票 -> NC
```

成本付款链路要讲清：

- 团期回团、项目完成、服务单完成后，成本如何进入财务；
- 供应商账单如何和业务成本匹配；
- 应付如何确认；
- 预付、已付、待付如何区分；
- 付款申请如何基于可申请金额发起；
- 审批通过后财务如何执行付款；
- 回单如何核销；
- 进项发票和 NC 如何承接。

### 第三条：结算链路

```text
团期结算/项目结算/渠道结算/门店结算 -> 财务复核 -> 差异处理 -> 结算确认 -> NC/报表
```

结算链路要讲清：

- 团期、项目、订单、服务单如何进入结算；
- 门店和渠道为什么需要月结对账；
- OTA 和代理账单如何与系统订单匹配；
- 财务如何处理收入、成本、退款、佣金、扣点、发票、回单差异；
- 结算确认后如何进入报表和 NC；
- 已结算后补账单、退款、汇率差异如何走结算调整。

## 五、场景一：订单产生应收

### 业务背景

销售订单确认后，系统不应该等财务重新录一张应收。订单金额、客户、合同、产品、团期、渠道、收款计划已经明确时，系统应自动形成财务应收。

### 上游业务动作

- 销售创建订单；
- 客户和联系人完整；
- 产品、团期、套餐、服务单或项目明确；
- 订单金额、优惠、补差、付款节点明确；
- 合同或收款计划确认。

### 系统自动形成什么

| 自动形成对象   | 说明                       |
| -------- | ------------------------ |
| 应收单      | 记录客户、来源订单、应收金额、已认金额、待收金额 |
| 应收计划     | 按订金、尾款、阶段款等节点形成          |
| 订单收款状态   | 待收款、部分收款、已收清             |
| 发票前置信息   | 客户抬头、可开金额、订单来源           |
| NC 待承接条件 | 认款或结算确认后进入 NC            |

### 财务要核什么

- 来源订单是否真实有效；
- 财务主体是否正确；
- 应收金额是否等于订单财务口径；
- 优惠、补差、退差是否已审批；
- 客户、渠道、门店对象是否准确；
- 币种、汇率、本币金额是否完整；
- 到期日和账龄是否正确。

### 系统入口

财务 > 应收管理

### 演示流程

1. 进入应收管理。
2. 按订单号、客户、渠道、团期或项目筛选。
3. 打开应收详情，查看来源订单和应收计划。
4. 核对应收金额、已认金额、待收金额、到期日和账龄。
5. 如果来源完整，确认应收或等待认款。
6. 如果金额、主体或客户信息异常，退回业务源单修正。

### 培训提醒

财务不要从订单截图或 Excel 手工录应收。应收应该来自订单、合同或项目阶段款。财务要核对来源和金额，不替销售补订单。

## 六、场景二：客户付款到账与认款核销

### 业务背景

客户付款到账后，系统要先形成到账流水，再通过自动识别或人工认款核销应收。到账不等于已收清，认款确认后才算财务核销。

### 到账来源

| 来源       | 系统处理方式               |
| -------- | -------------------- |
| 银行流水     | 银行接口同步或 Excel/CSV 导入 |
| 银联/POS   | 接口同步或上传银联水单、POS 清算文件 |
| 微信/支付宝   | 支付平台自动同步             |
| 门店收款     | 门店上传水单或总部导入          |
| OTA/渠道回款 | 平台账单、渠道对账单、回款流水匹配    |
| 线下特殊收款   | 财务登记流水，但仍需识别和认款      |

### 系统自动形成什么

| 自动形成对象 | 说明                     |
| ------ | ---------------------- |
| 到账流水   | 记录流水号、到账时间、付款方、收款账户、金额 |
| 识别结果   | 系统根据付款方、金额、订单号、附言识别用途  |
| 认款候选   | 自动匹配可能的应收单、订单或预存账户     |
| 认款单    | 财务确认一笔流水如何核销应收         |
| 收款凭证   | 认款确认后生成                |
| NC 任务  | 收款凭证形成后进入 NC 待推送       |

### 系统入口

财务 > 到账流水  
财务 > 收款认款

### 演示流程：到账流水

1. 进入到账流水。
2. 演示银行流水导入或银联水单上传。
3. 查看导入预览，重点看流水号、到账时间、付款方、金额、重复校验。
4. 确认导入后，流水进入待识别或已识别状态。
5. 打开某条流水，查看系统识别依据。

### 演示流程：收款认款

1. 从已识别流水进入收款认款。
2. 查看系统匹配的应收单和订单。
3. 对金额、付款方、客户、订单进行核对。
4. 一笔流水对应多张订单时，演示拆分认款。
5. 多笔流水对应一张订单时，演示分次核销。
6. 到账金额大于应收时，差额转预收。
7. 确认认款后，应收状态更新，生成收款凭证和 NC 任务。

### 财务要核什么

- 流水是否重复；
- 付款方是否与客户、门店或渠道一致；
- 收款账户是否属于正确财务主体；
- 到账金额是否等于应收、部分应收或多笔应收合计；
- 银联/POS 水单和实际到账是否有清算差异；
- 超收、短收、拆分、合并是否有明确处理。

### 常见异常

| 异常         | 财务处理        |
| ---------- | ----------- |
| 付款方能匹配多个客户 | 人工确认客户和订单   |
| 金额能匹配多张应收  | 拆分认款        |
| 到账金额大于应收   | 差额转预收       |
| 到账金额小于应收   | 部分认款，保留待收   |
| 认错款        | 冲正认款，不删除原凭证 |
| 水单信息不完整    | 退回门店或补充水单信息 |

## 七、场景三：退款和售后反冲

### 业务背景

售后退款由销售或客服发起，财务负责执行和入账。售后审批通过不等于已经退款，财务执行成功并核销回单后，退款才完成。

### 上游业务动作

- 销售发起退款、退差、转团、改期、改舱、改铺；
- 售后规则和可退金额计算完成；
- 必要审批通过；
- 退款账户和客户信息完整。

### 系统自动形成什么

| 自动形成对象  | 说明           |
| ------- | ------------ |
| 退款单     | 审批通过后进入财务待执行 |
| 应收冲减    | 未收部分冲减应收     |
| 退款付款任务  | 已收部分进入退款执行   |
| 发票红冲任务  | 已开票订单退款后触发   |
| NC 退款任务 | 退款执行完成后生成    |

### 系统入口

财务 > 退款执行  
财务 > 转款核销  
财务 > 发票管理

### 演示流程

1. 打开退款执行列表。
2. 查看退款单来源订单、客户、退款原因、可退金额、本次退款。
3. 核对原收款和应收冲减。
4. 核对退款账户。
5. 执行退款。
6. 上传或同步退款回单。
7. 如果原订单已开票，进入发票管理处理红冲。
8. 退款完成后生成退款凭证和 NC 任务。

### 财务要核什么

- 退款是否已经审批；
- 原订单是否已收款；
- 可退金额是否准确；
- 未收部分是否先冲应收，已收部分才退款；
- 退款账户是否与客户或原付款方一致；
- 已开票是否触发红冲；
- 退款完成是否回写订单和 NC。

### 培训提醒

财务不处理“客户为什么退”，也不修改库存释放规则。财务处理的是可退金额、退款账户、付款执行、回单、发票红冲和凭证。

## 八、场景四：团期/项目成本形成应付

### 业务背景

团期回团、项目完成或服务单完成后，业务侧会沉淀成本事实。财务不能凭口头、微信、Excel 或供应商一句话付款，必须看到成本来源、供应商账单、费用项目、附件和业务对象。

### 上游业务动作

- 计调确认团期实际成本；
- 项目经理提交项目成本；
- 自由行采购批次完成供应商对账；
- 服务单确认代垫或服务成本；
- 供应商账单上传或导入；
- 业务侧提交结算申请或成本确认。

### 系统自动形成什么

| 自动形成对象 | 说明                 |
| ------ | ------------------ |
| 成本确认记录 | 业务侧确认实际成本          |
| 供应商账单  | 供应商账单进入识别和匹配       |
| 应付草稿   | 成本和账单匹配后形成应付基础     |
| 应付单    | 财务确认后形成付款义务        |
| 进项发票任务 | 账单或应付确认后进入发票管理     |
| 结算承接   | 成本进入团期、项目、订单或服务单结算 |

### 系统入口

财务 > 供应商账单  
财务 > 应付管理

### 演示流程：供应商账单

1. 进入供应商账单。
2. 查看账单来源、供应商、关联团期/项目、账单金额、币种和汇率。
3. 系统匹配业务成本项。
4. 匹配成功则进入应付确认。
5. 无业务对象则进入待挂接。
6. 金额不一致则进入差异处理。

### 演示流程：应付确认

1. 进入应付管理。
2. 查看应付来源、供应商、费用项目、原币应付、汇率、本币金额。
3. 核对已付、待付、预付冲抵和发票风险。
4. 确认应付。
5. 应付确认后允许进入付款申请。

### 财务要核什么

- 供应商是否准确；
- 账单是否挂接到正确团期、项目、采购批次或服务单；
- 成本是否实际发生，不是预估成本；
- 金额、币种、汇率是否准确；
- 供应商账单和业务成本是否一致；
- 发票或附件是否齐全；
- 是否存在预付需要冲抵。

### 培训提醒

预估成本不是应付。供应商账单确认也不等于已经付款。财务确认应付后，才进入付款申请和付款执行。

## 九、场景五：付款申请、审批、付款执行

### 业务背景

付款链路必须讲清四个动作：业务发起付款申请、审批中心审批、财务执行付款、回单核销。付款申请通过不等于已付款。

### 上游业务动作

- 业务侧基于成本、账单或合同节点发起付款申请；
- 财务复核付款依据；
- 审批中心审批付款申请；
- 审批通过后进入出纳付款执行。

### 系统自动形成什么

| 自动形成对象  | 说明                     |
| ------- | ---------------------- |
| 付款申请单   | 记录付款对象、依据、可申请金额、本次申请金额 |
| 审批任务    | 按金额、主体、供应商、付款类型触发      |
| 待执行付款任务 | 审批通过后进入出纳付款            |
| 付款单     | 付款执行成功后生成              |
| 回单核销任务  | 付款完成后等待回单确认            |
| NC 付款任务 | 回单核销后进入 NC             |

### 系统入口

财务 > 付款申请  
财务 > 付款执行  
财务 > 回单核销  
财务 > 付款退回

### 必须讲清的金额口径

| 金额     | 含义               |
| ------ | ---------------- |
| 应付金额   | 已确认应付总额          |
| 已申请金额  | 历史已发起付款申请金额      |
| 可申请金额  | 当前还能申请的余额        |
| 本次申请金额 | 本次实际申请付款金额       |
| 已付款金额  | 出纳已经执行成功并回写的金额   |
| 待付金额   | 应付扣除已付款和预付冲抵后的余额 |

### 演示流程

1. 进入付款申请。
2. 查看付款对象、付款依据、关联团期/项目、可申请金额和本次申请金额。
3. 核对供应商账单、应付单、合同节点、预付冲抵和附件。
4. 复核通过后进入审批。
5. 审批通过后进入付款执行。
6. 出纳核对付款账户、收款账户、账户余额、币种和付款方式。
7. 执行付款。
8. 上传或同步回单。
9. 回单核销后更新应付状态，生成 NC 付款任务。

### 财务要核什么

- 是否超过可申请金额；
- 是否重复申请；
- 供应商账户是否有效；
- 付款账户余额是否足够；
- 外币付款是否有汇率；
- 预付是否已正确冲抵；
- 回单金额是否与付款单一致。

### 常见异常

| 异常       | 财务处理          |
| -------- | ------------- |
| 付款申请附件不足 | 退回业务补依据       |
| 超额申请付款   | 阻断或进入特殊审批     |
| 供应商账户异常  | 退回补账户信息       |
| 银行付款失败   | 进入付款退回或失败处理   |
| 回单金额不一致  | 进入回单差异处理      |
| 已审批未付款   | 留在付款执行，不更新已付款 |

## 十、场景六：团期/项目/渠道/门店结算

### 业务背景

结算是财务对一个业务对象的经营结果进行确认。团期、项目、自由行订单、单项服务、门店、渠道、OTA 都有可能进入结算。结算不是业务补录，也不是单纯报表，它是财务把收入、成本、收款、退款、应付、付款、发票、回单和毛利统一核对。

### 结算类型

| 结算类型      | 核心核对内容                 |
| --------- | ---------------------- |
| 团期结算      | 团期收入、实际成本、退款、应付、付款、毛利  |
| 项目结算      | 锁定报价、阶段收款、项目成本、变更、毛利   |
| 自由行订单结算   | 订单收入、采购批次成本、供应商对账、订单利润 |
| 单项服务结算    | 服务费收入、代垫款、服务成本、退款扣损    |
| 门店结算      | 门店订单、收款、水单、预存扣款、退款、差异  |
| 渠道/OTA 结算 | 渠道订单、平台账单、佣金扣点、退款、应回款  |

### 系统入口

财务 > 业务结算  
财务 > 门店对账  
财务 > 渠道结算  
财务 > 结算调整

### 演示流程：业务结算

1. 进入业务结算。
2. 按团期、项目、订单或服务单筛选。
3. 查看结算摘要：收入、成本、已收、待收、已付、待付、毛利。
4. 核对收入明细：应收、已认款、退款、优惠、待收。
5. 核对成本明细：供应商、费用项目、应付、预付冲抵、已付、待付。
6. 核对发票和回单。
7. 处理差异。
8. 确认结算。
9. 结算确认后进入报表和 NC。

### 演示流程：门店月结

1. 进入门店对账。
2. 选择门店和对账周期。
3. 系统汇总门店订单、门店收款、银联/POS 水单、预存充值、预存扣款、退款。
4. 财务查看差异清单。
5. 处理缺流水、预存未确认、退款未执行、订单状态不一致等问题。
6. 确认门店对账。
7. 结果进入结算、预存调整、应收或 NC。

### 演示流程：渠道/OTA 对账

1. 导入或同步渠道/OTA 对账单。
2. 系统按订单号、金额、客户、日期匹配系统订单。
3. 自动计算佣金、扣点、退款折减和应回款。
4. 财务处理平台账单和系统订单差异。
5. 确认渠道结算。
6. 结果进入应收、认款、佣金应付、结算和 NC。

### 财务要核什么

- 收入是否来自已确认应收和认款；
- 成本是否来自已确认成本或应付；
- 退款是否已执行；
- 预收、预存、预付是否正确抵扣；
- 门店或渠道账单是否和系统订单一致；
- 发票、回单是否存在风险；
- 毛利是否异常；
- NC 是否可推送。

### 培训提醒

以前月结对账靠 Excel 汇总，系统目标是自动汇总订单、收款、退款、预存、佣金和扣点，财务只处理差异。

## 十一、场景七：发票与回单

### 业务背景

发票和回单是财务最关心的证据链。收入侧要看销项开票，成本付款侧要看进项收票和付款回单。发票、回单状态会影响结算风险、NC 和报表口径。

### 发票链路

```text
应收/收款/结算/应付/付款/退款
-> 发票任务
-> 开票/收票/验真/红冲
-> 税务台账
-> NC
```

### 回单链路

```text
付款执行/退款执行
-> 银行回单/支付回执
-> 回单核销
-> 付款完成/退款完成
-> NC
```

### 系统入口

财务 > 发票管理  
财务 > 回单核销

### 自动开票如何讲

自动开票不是财务不用管，而是系统根据应收、客户抬头、税号、可开金额、税率和发票类型自动生成开票任务。税务会计核对后，对接电子发票或税控平台开票，结果回写系统。

### 演示流程：发票

1. 进入发票管理。
2. 按待开票、待收票、待验真、待红冲、异常处理中筛选。
3. 打开发票详情，查看业务来源。
4. 核对开票/收票对象、金额、税率、抬头、税号。
5. 执行开票、登记收票、验真或红冲。
6. 发票完成后生成税务台账和 NC 任务。

### 演示流程：回单

1. 付款或退款执行完成后，进入回单核销。
2. 上传或同步银行回单。
3. 系统匹配付款单或退款单。
4. 财务核对对象、金额、账户、币种。
5. 回单核销后，付款或退款状态完成。
6. 生成 NC 凭证任务。

### 常见异常

| 异常         | 财务处理        |
| ---------- | ----------- |
| 发票抬头缺失     | 退回销售或客户档案补充 |
| 开票金额超过可开金额 | 阻断或拆分开票     |
| 退款已执行但未红冲  | 生成红冲任务      |
| 进项票金额大于应付  | 进入应付差异处理    |
| 回单金额不一致    | 进入回单差异处理    |
| 外币回单汇率不一致  | 进入汇率差异处理    |

## 十二、场景八：NC 推送与异常处理

### 业务背景

NC 是最终入账承接系统。ERP 中已经确认的财务事件，会生成 NC 凭证任务。NC 推送失败不代表业务失败，也不能回滚业务状态。财务要修复失败原因后重新推送。

### NC 来源

| 来源   | 何时生成 NC 任务     |
| ---- | -------------- |
| 收款凭证 | 认款确认后          |
| 退款凭证 | 退款执行完成后        |
| 应付确认 | 应付单确认后，按规则生成   |
| 付款凭证 | 付款执行并回单核销后     |
| 发票任务 | 开票、收票、红冲或验真完成后 |
| 结算单  | 结算确认后          |
| 结算调整 | 调整生效后          |
| 汇兑差异 | 外币结算或关账确认后     |

### 系统入口

财务 > NC 推送

### 演示流程

1. 进入 NC 推送。
2. 查看待推送、失败和日志。
3. 打开一张 NC 凭证任务。
4. 查看来源单据、金额、科目、辅助核算、客户/供应商、币种、汇率。
5. 对待推送任务执行推送。
6. 对失败任务查看失败原因。
7. 根据失败原因修复：科目、辅助核算、主体、币种、发票号、客户供应商档案、凭证字段。
8. 修复后重新推送。
9. 成功后保存 NC 返回号和日志。

### 财务要核什么

- 来源单据是否已生效；
- 科目和辅助核算是否完整；
- 客户、供应商、门店、渠道是否有 NC 对照；
- 金额、币种、汇率是否一致；
- 发票号、回单号是否满足凭证规则；
- 推送失败是否有明确修复路径。

### 培训提醒

NC 页面不能直接修改业务源单。NC 失败只影响凭证推送状态，不代表订单、收款、付款或结算失败。修复应回到正确的源单或配置项。

## 十三、财务如何检查上游业务数据质量

财务培训要让财务明白：系统不是让财务替业务补数据，而是让财务更早发现数据缺口，并通过退回、卡控、异常提示倒逼业务录准。

| 上游问题      | 财务会在哪里发现    | 处理方式         |
| --------- | ----------- | ------------ |
| 订单客户信息不完整 | 应收、开票       | 退回销售补客户和抬头   |
| 订单金额或优惠异常 | 应收、结算       | 退回销售或审批改价    |
| 合同或收款计划缺失 | 应收、认款       | 退回销售补合同/节点   |
| 名单或资源未确认  | 成本、结算       | 退回履约补执行结果    |
| 成本缺供应商    | 应付          | 退回业务补供应商     |
| 供应商账单未挂接  | 供应商账单       | 挂接团期、项目或采购批次 |
| 付款账户缺失    | 付款申请、付款执行   | 退回补供应商账户     |
| 发票金额不一致   | 发票管理        | 进入发票异常或应付差异  |
| 外币汇率未维护   | 应收、应付、付款、结算 | 财务维护或锁定汇率    |
| NC 辅助核算缺失 | NC 推送       | 补科目或辅助核算映射   |

培训时可以这样讲：

“财务不是靠微信群追业务资料，而是在系统里看到哪个对象缺什么、影响哪个财务动作，然后按对象退回或卡控。”

## 十四、财务日常工作台怎么用

这里的“工作台”不是一定要有一个单独页面，而是财务每天应该按角色和状态筛选处理自己的待办。

### 应收会计每天看什么

1. 到账流水：待识别、已识别待认款、银联/POS 水单差异。
2. 收款认款：待复核、差异中、超收待处理。
3. 应收管理：到期、逾期、部分收款、异常应收。
4. 预收管理：可用余额、冻结、退款中。
5. 转款核销：待财务确认、差异处理中。

### 出纳每天看什么

1. 资金池：余额差异、待处理资金、付款占用。
2. 付款执行：待执行、执行失败、待回单。
3. 退款执行：待退款、执行中、退款退回。
4. 回单核销：待匹配、待核销、金额差异。
5. 付款退回：待匹配原付款、待确认退回。

### 应付会计每天看什么

1. 供应商账单：待识别、待挂接、待核对、差异中。
2. 应付管理：待核对、有差异、可付款、付款中。
3. 付款申请：待复核、审批中、已审批待转付款。
4. 预付管理：待分摊、可冲抵、长期未冲抵。
5. 发票管理：进项待收票、待验真、票据差异。

### 税务会计每天看什么

1. 发票管理：待开票、待收票、待验真、待红冲。
2. 自动开票任务：抬头、税号、金额、税率是否正确。
3. 税控/电子发票同步失败。
4. 退款触发的红冲任务。
5. 发票 NC 状态。

### 结算和总账每天看什么

1. 业务结算：待核对、差异中、待确认。
2. 门店对账：本月待确认、差异中。
3. 渠道结算：平台账单待匹配、佣金扣点差异。
4. NC 推送：待推送、失败、待重推。
5. 期间关账：预检缺口、异常清单、关账状态。

## 十五、月结、对账、报表和管理看板

### 月结对账

财务月结不是月底手工汇总 Excel，而是系统按已发生业务对象自动汇总，财务处理差异。

| 月结对象    | 系统自动汇总               | 财务处理             |
| ------- | -------------------- | ---------------- |
| 门店月结    | 门店订单、收款、水单、预存扣款、退款   | 核差异、确认对账、形成结算    |
| 渠道月结    | 渠道订单、平台账单、佣金扣点、退款、回款 | 核平台差异、确认应回款或应付佣金 |
| OTA 对账  | OTA 订单、平台账单、扣点、退款    | 自动匹配订单，财务处理未匹配   |
| 供应商对账   | 供应商账单、团期成本、项目成本、应付   | 核金额差异，确认应付       |
| 团期/项目结算 | 应收、已收、退款、成本、应付、已付、发票 | 核毛利和差异，确认结算      |

### 财务报表

报表不是财务重新统计，而是系统从已生效单据自动沉淀。

| 报表       | 来源对象            | 作用           |
| -------- | --------------- | ------------ |
| 应收余额表    | 应收、认款、退款        | 看客户、门店、渠道待收  |
| 收款认款明细   | 到账流水、认款、收款凭证    | 看钱从哪里来、认到哪里  |
| 未认款流水表   | 到账流水            | 看待识别和异常流水    |
| 预收/预存余额表 | 预收、预存账户         | 看余额、冻结、使用    |
| 应付余额表    | 应付、付款、预付冲抵      | 看供应商待付       |
| 付款执行表    | 付款申请、付款单、回单     | 看付款进度和回单状态   |
| 团期/项目毛利表 | 结算单、收入、成本       | 看经营结果        |
| 门店经营报表   | 门店订单、收款、预存、结算   | 看门店经营和月结     |
| 渠道结算报表   | 渠道订单、平台账单、佣金、回款 | 看渠道对账和应回款    |
| 发票统计表    | 发票任务、红冲、税控同步    | 看开票、收票和异常    |
| 多币种汇率差异表 | 外币应收、应付、付款、结算   | 看汇兑损益        |
| NC 推送报表  | NC 凭证任务和日志      | 看推送成功、失败、待处理 |

### 管理看板

财务管理层更关心趋势和风险：

- 本月应收、已收、待收；
- 本月应付、已付、待付；
- 未认款流水金额；
- 门店和渠道差异金额；
- 供应商预付余额和账龄；
- 团期/项目毛利异常；
- 发票异常；
- NC 推送失败；
- 关账预检缺口。

培训时要强调：看板和报表可以穿透到源单，但不在报表里直接修改业务数据。

## 十六、常见异常和处理口径

| 异常       | 不能怎么做      | 正确处理                     |
| -------- | ---------- | ------------------------ |
| 应收金额不一致  | 财务手工改应收    | 回到订单、合同、优惠或补差源头修正        |
| 到账流水无法识别 | 随便挂某个订单    | 查付款方、附言、水单，必要时待识别        |
| 认错款      | 删除认款记录     | 做认款冲正，保留原凭证              |
| 超收       | 随便冲其他订单    | 转预收或挂账，后续按规则使用           |
| 退款审批通过   | 认为已退款      | 财务执行退款并回单后才完成            |
| 成本依据缺失   | 财务补成本      | 退回履约或计调补供应商账单和附件         |
| 供应商账单差异  | 直接按供应商金额付款 | 做差异处理，必要时补审批             |
| 付款申请通过   | 认为已付款      | 出纳执行成功并回单核销后才算付款完成       |
| 预付已付款    | 直接当成本      | 后续冲抵实际应付                 |
| 发票金额不一致  | 强行开票或收票    | 进入发票异常或应收/应付差异处理         |
| 汇率缺失     | 随便按当前汇率    | 按财务规则选择业务日、账单日、付款日或结算日汇率 |
| NC 推送失败  | 回滚业务单据     | 修复科目、辅助核算、对象、币种或凭证字段后重推  |
| 报表数不对    | 直接改报表      | 穿透源单，检查单据状态和生效口径         |

## 十七、结课检查：财务要能说清这些问题

培训结束前，让财务同事用自己的话回答：

1. 为什么订单确认后会自动生成应收。
2. 到账流水、认款单、收款凭证有什么区别。
3. 银联/POS/门店水单上传后，系统如何进入认款链路。
4. 一笔流水对应多张订单时，系统怎么拆分认款。
5. 超收为什么进入预收，而不是随意冲其他订单。
6. 售后退款审批通过后，财务还要做哪些动作。
7. 团期回团后，成本如何变成应付。
8. 供应商账单和业务成本金额不一致时怎么处理。
9. 付款申请、审批通过、付款执行、回单核销有什么区别。
10. 预付为什么不能直接当成本。
11. 自动开票任务从哪里来，退款后为什么会触发红冲。
12. 多币种业务为什么要保留原币、汇率、本币和汇率来源。
13. 门店月结和渠道/OTA 对账分别核什么。
14. 财务报表为什么必须从已生效单据自动生成。
15. NC 推送失败时应该修什么，不应该做什么。

如果财务能说清这些问题，说明这场培训没有停留在“财务菜单怎么点”，而是真正理解了 ERP 如何把旅游业务数据转化成财务对象、资金结果、票据结果、结算结果、NC 凭证和管理报表。
