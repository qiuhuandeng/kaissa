本文是第七步“校准菜单和页面归属”的验收物。它把现有商户端页面按业务对象、数据流和职责边界重新归属，避免后续页面因为找不到归属回退到工作台，或把供应商、渠道、财务、售后、团期等对象放错模块。

## 1. 第七步目标

菜单归属必须服务总目标：

```text
让凯撒旅游 SaaS 的产品对象、库存对象、订单对象、成本对象、收付对象、结算对象、NC 凭证对象形成一条可追溯的数据链路。
```

页面归属只看业务对象，不看岗位名称。

每个页面必须能回答：

1. 当前页面维护哪个业务对象。
2. 这个对象从哪里来。
3. 这个对象流向哪里。
4. 是否影响库存、订单、成本、收款、付款、退款、结算。
5. 是否触发财务单据。
6. 是否进入 NC 推送或税务台账。

## 2. 本次代码校准结果

本次已修改 [shared/nav-merchant.js](../shared/nav-merchant.js)，完成以下校准：

| 校准点 | 调整前 | 调整后 | 业务原因 |
|---|---|---|---|
| 别名标准 | 资源一级叶子 | 资源 / 基础资源 / 别名标准 | 别名属于基础资源主数据 |
| 售后处理 | 订单中心单一叶子 | 订单中心 / 售后处理 / 售后总览、转团/改期、退款申请 | 售后总览、变更申请、退款申请是订单售后对象，不是财务退款执行 |
| 渠道对象命名 | 分销渠道、同业代理 | 分销商管理、代理商管理 | 分销商、代理商属于渠道对象，不能复用供应商口径 |
| 资金池 | 财务报表里的一个统计项 | 财务一级“资金池”，报表内保留“资金池余额” | 资金池是资金节点和流水对象，不只是报表 |
| 系统设置命名 | 组织管理、员工管理、角色授权、业务参数 | 组织架构、角色权限、员工账号、基础参数 | 与全局菜单固定命名对齐 |
| 竞品价格页 | 未被菜单或 pageOwners 覆盖 | 归属到产品中心 / 定价策略 | 竞品价格服务价格策略，不归 AI 竞品分析主功能 |
| 产品中心分组 | 产品市场、产品管理、团期管理、单团自组散列 | 外采产品 / 产品市场、代理产品；自营产品 / 产品管理、单团自组、团期管控、团期结算 | 按外采供给和自营经营两条主线组织入口 |

## 3. 商户端一级菜单归属

| 一级菜单 | 管理对象 | 关键边界 |
|---|---|---|
| 工作 | 工作台、待办、消息 | 不承载业务主对象 |
| 资源 | POI、酒店、餐厅、车型、航线、锁位、邮轮资源、专列资源、供应商、领队 | 只维护可复用主数据，不维护可售产品结构 |
| 产品中心 | 线路、方案、团期、自由行套餐、单项服务、外采包装、定价、渠道授权 | 管产品结构、库存结构和团期执行入口 |
| 订单中心 | 报价、意向、销售订单、合同、收款认领、售后申请 | 管客户交易和售后申请，不执行财务退款 |
| 门店中心 | 门店、顾问、门店资金节点、门店对账 | 管自有销售网络和门店经营对象 |
| 渠道中心 | OTA、分销商、代理商、佣金规则、同步日志 | 管外部销售渠道，不管理供应商 |
| 会员 | 会员、联系人、跟进、标签、分群 | 管会员主数据和会员经营 |
| 财务 | 应收、认款、收款、应付、付款、退款执行、结算、发票、资金池、NC、报表 | 管财务单据、执行和报表，不替代业务申请 |
| 审批 | 审批单、审批配置、抄送、审批总览 | 审批只控制业务单据是否生效 |
| AI | 线路拆解、竞品分析、签证审查、出行助理、模型能力 | AI 是能力入口，不替代业务对象 |
| 系统设置 | 组织、角色、员工、接口、参数、通知、审计 | 管系统配置，不承载交易对象 |

## 10. 新增页面归属规则

新增页面必须先判断对象，再决定菜单：

1. 可复用主数据进入资源。
2. 长期可售旅游产品进入产品中心。
3. 参团游、邮轮、专列、自由行、研学、单项服务进入 自营产品 / 产品管理。
4. 可外采线路产品进入 外采产品 / 产品市场；代理入库后进入 外采产品 / 代理产品。
5. 团期、成本、出团执行、回团结算进入团期管理。
6. 销售订单、合同、售后申请进入订单中心。
7. 退款执行、付款执行、应收、应付、结算、发票、NC 进入财务。
8. 分销商、代理商、OTA 进入渠道中心。
9. 供应商进入资源，不进入渠道中心。
10. 审批详情归审批中心，审批通过后回写原业务对象。

新增二级页、详情页、新建页、编辑页必须在 `shared/nav-merchant.js` 的 `pageOwners` 中配置归属，除非它已经是左侧菜单直达叶子页。

## 11. 第七阶段验收标准
