# 单团自组实施基线

更新时间：2026-07-07

本文用于固定凯撒旅游 SaaS 单团自组改造的业务目标、对象边界和页面实施顺序。后续每次继续改造单团、独立成团、项目报价、专属团期、订单合同、执行成本或结算承接页面前，必须先读取本文，再判断当前页面在全链路中的位置，避免只沿着上一个页面的局部按钮或字段继续堆功能。

## 1. 业务目标

单团自组要解决的是客户专属团队从需求、报价、锁定、合同订单、专属团期、执行成本到项目结算的业务承接。

单团不是一个新的标准线路类型，也不是资源模块的动态库存入口。它是产品中心下的项目型成团入口，主入口保留在：

```text
产品中心 -> 单团项目
```

菜单和页面主标题统一使用“单团项目”；“单团自组”只作为业务场景口径保留在说明中。

## 2. 两条业务路径

### 2.1 引用标准线路独立成团

适用于客户选择已有成熟线路，只要求独立成团、专属日期、专属人数、专属报价或局部服务调整。

```text
旅游线路/方案 -> 创建专属团期 -> 团体订单 -> 出团执行 -> 团期成本 -> 回团结算
```

关键边界：

- 不新增“单团线路类型”。
- 独立成团是团期的成团方式，不是线路类型。
- 专属团期必须关联客户或单团项目。
- 专属团期不进入公开渠道售卖。
- 团期仍承接名单、资料、执行、成本和回团结算。

### 2.2 新建定制单团项目

适用于客户只有需求，行程、资源、服务标准、会议活动、预算和报价都需要重新组合。

```text
单团需求 -> 报价版本 -> 锁定报价 -> 单团项目 -> 项目团期 -> 合同订单 -> 执行成本 -> 项目结算
```

关键边界：

- 报价前是需求。
- 客户确认后是项目。
- 执行时承接到专属团期或项目团期。
- 收款时承接到订单和应收计划。
- 结算时承接到项目结算单。
- 后续可支持“沉淀为标准线路/方案”，但不是第一轮主流程。

## 3. 对象边界

| 对象 | 维护位置 | 说明 |
|---|---|---|
| 单团需求 | 单团自组列表/详情 | 客户、联系人、人数、日期、目的地、预算、服务要求、会议活动要求 |
| 报价版本 | 单团详情 | A/B/C 多版本报价，包含行程、服务、预算成本、售价和预计毛利 |
| 锁定报价 | 单团详情 | 客户确认后的只读快照，后续金额或资源变化走变更记录 |
| 单团项目 | 单团详情 | 客户确认后的项目主对象，用于承接合同、订单、执行和结算 |
| 专属团期/项目团期 | 履约中心 > 团期管控 | 用于名单、资源确认、执行、成本和回团结算 |
| 团体订单 | 销售中心 | 用于合同、收款计划、游客名单、售后和退款 |
| 项目成本 | 团期成本/单团详情承接入口 | 业务侧确认成本，财务侧承接应付和付款 |
| 项目结算单 | 财务 > 业务结算 | 财务确认收入、退款、成本、应付、毛利和 NC |

## 4. 页面责任边界

单团主页面负责让业务人员理解和推进流程，不负责深度财务执行。

单团详情页可以展示：

- 报价版本和锁定报价。
- 合同、订单、专属团期、收款计划、成本和结算的承接状态。
- 进入订单、团期、财务页面的入口。
- 关键数据缺口，如报价未锁定、合同未生成、名单缺失、资源未确认、成本未确认。

单团详情页不做：

- 财务认款。
- 付款执行。
- 发票认证。
- NC 推送。
- 跨页面真实自动回写。
- 自动生成完整下游单据闭环。

## 5. 页面实施卡模板

每次改造页面前，必须先按以下模板确认：

| 项 | 必答 |
|---|---|
| 页面位置 | 当前页面位于单团链路哪一段 |
| 使用角色 | 销售、产品/计调、团控、财务、审批人谁为主 |
| 上游来源 | 从需求、线路、报价、团期、订单哪个对象来 |
| 当前对象 | 当前页面维护的唯一主对象是什么 |
| 高频任务 | 用户进入页面最常做什么 |
| 下游承接 | 完成后流向订单、团期、成本、财务还是审批 |
| 不做事项 | 当前页面不处理哪些下游动作 |

## 6. 第一轮实施范围

第一轮只处理单团主入口，不做全系统闭环。

必改页面：

- `merchant/tour/product-custom-list.html`
- `merchant/tour/product-custom-detail.html`

第一轮目标：

- 列表统一为“单团项目”入口。
- 支持“新建定制需求”和“引用线路创建独立成团”两个入口。
- 列表操作列稳定，不用当前待办生成不同按钮。
- 详情页按需求、报价、锁定、合同订单、执行资源、成本、收款结算、变更记录组织。
- 财务和团期只做承接入口，不做深闭环。

## 7. 后续实施顺序

1. 单团列表和详情。
2. 标准线路/方案发起独立成团。
3. 新建团期支持独立成团和客户/项目关联。
4. 团期列表和详情识别专属团期。
5. 订单、合同、意向订单承接单团来源。
6. 执行、成本、结算页面识别单团项目来源。
7. 财务、审批、客户详情补充反查和来源筛选。

## 8. 防跑偏规则

后续任何页面改造，如果发现页面在尝试把单团做成标准线路类型、资源日常操作入口、财务深闭环页面或 NC 推送入口，必须先收敛为业务承接入口，再判断是否需要修改下游页面。

一句话基线：

```text
单团主入口管需求、报价、锁定和项目推进；团期管执行；订单管合同收款；财务管应收应付结算和 NC。
```
