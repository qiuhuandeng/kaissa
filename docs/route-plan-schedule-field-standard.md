# 产品字段分层标准

更新时间：2026-06-29

关联文档：

- [产品域与业财流程总模型基线](product-domain-finance-flow.md)
- [产品域对象字典](product-domain-object-dictionary.md)
- [产品类型矩阵](product-type-matrix.md)

本文用于固定凯撒旅游 SaaS 的字段分层边界。后续线路编辑页、方案编辑页、团期新建/批量开排页、自由行套餐页、单项服务页、MICE 项目页、外采产品页、订单页和财务页，都必须按本文判断字段应该放在哪一层。

## 1. 字段分层总规则

字段按业务事实发生的位置分层：

| 层级 | 维护内容 | 不能维护 |
|---|---|---|
| 资源层 | 可复用基础资料，如 POI、酒店、餐厅、车型、航线、船只、专列线路、供应商、领队 | 某个产品的售卖价格、团期库存、订单收款 |
| 线路层 | 长期稳定的售卖对象，回答“卖什么” | 具体出发日期、当期库存、当期成交价、当期成本 |
| 方案层 | 线路下的服务结构，回答“怎么组织服务” | 某一天卖多少、剩多少、收了多少钱 |
| 团期层 | 某方案在具体日期的销售和执行实例，回答“哪天卖、卖多少钱、谁执行” | 完整重复行程、通用费用政策、长期素材 |
| 出行日期库存层 | 自由行套餐某日期的价格、房量、可售量 | 团期名单、领队、成团状态 |
| 订单层 | 客户购买事实、游客、合同、应收金额 | 供应商主数据、长期产品规则 |
| 执行成本层 | 执行任务、回团、实际成本、供应商账单 | 客户应收、收款流水 |
| 财务层 | 应收、认款、收款、退款、结算、应付、付款、发票、NC | 产品内容和行程结构 |

通用禁止规则：

1. 线路层不维护具体出发日期、当期价格和库存。
2. 方案层不维护团期库存和团期成交价。
3. 团期层不重复录入完整行程，只引用方案并记录当期变化。
4. 自由行不进入团期层，使用出行日期价格库存。
5. 单项服务不进入线路、方案、团期三层模型。
6. 外采不是产品类型，只叠加采购价、包装、配额和差额核算字段。
7. 财务字段必须有业务来源对象，不能孤立录入。

## 2. 标准线路共通模型

跟团、邮轮、专列、研学统一采用：

```text
线路 -> 方案 -> 团期 -> 订单 -> 执行/成本 -> 结算 -> NC
```

共通命名：

| 概念 | 统一命名 | 说明 |
|---|---|---|
| 产品类型 | 线路类型 | 跟团、邮轮、专列、研学、自由行等 |
| 产品名称 | 线路名称 | 标准线路长期售卖名称 |
| 方案 | 线路方案 | 同一线路下的不同服务结构 |
| 日期售卖单元 | 团期 | 仅适用于跟团、邮轮、专列、研学 |
| 自由行日期单元 | 出行日期价格库存 | 不叫团期 |
| 单项服务订单 | 服务单 | 不叫线路订单 |

## 14. 页面字段验收规则

每个页面交付前必须检查：

1. 页面主对象是否明确。
2. 字段是否放在正确层级。
3. 标准线路页面是否遵守线路、方案、团期三层边界。
4. 邮轮是否按舱型结构维护库存和价格。
5. 专列是否按铺位/座位结构维护库存和价格。
6. 研学是否包含年级、主题、课程、基地、导师、安全和保险字段。
7. 自由行是否使用套餐和出行日期库存，不出现团期。
8. 单项服务是否使用服务项目和服务单，不出现线路。
9. MICE 是否使用需求、报价版本、锁定报价和项目执行。
10. 外采是否只叠加采购价、包装、配额和差额核算。
11. 订单页是否只维护购买事实，不维护供应商应付。
12. 财务页是否能追溯业务来源对象。

## 15. 第四阶段验收标准
