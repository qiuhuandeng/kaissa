本文定义凯撒旅游 SaaS 的营销对象、优惠计算、承担分摊和页面投放边界。后续营销中心、订单优惠明细、门店与分销结算、小程序页面配置、审批和分析报表必须遵循本文，不得各自重复定义优惠规则。

## 1. 营销域定位与边界

营销域服务总部营销运营、区域运营、门店、渠道经理和小程序运营，目标是将可销售的旅游产品、团期和服务以受控优惠方式投放给直客或分销商，并将优惠结果完整承接到订单、售后和结算。

营销域处在以下链路中：

```text
产品/方案/团期/出行日期/服务项目
  -> 营销活动或优惠券
  -> 发券任务或小程序页面投放
  -> 订单优惠计算
  -> 应收、退款、门店/分销结算、营销分析
```

营销域不是独立价格体系，也不是财务模块。

- 定价策略仍维护基础销售价、渠道价和最低销售价。
- 营销活动和优惠券只在基础销售价之上计算优惠，不允许绕过定价策略直接维护销售价。
- 佣金政策仍维护门店、分销商与集团之间的佣金规则；营销域只提供每笔优惠由谁承担的结果。
- 产品、方案、团期、舱型、铺位、自由行出行日期、服务项目和可售渠道仍由产品中心维护。
- 发券、领券、使用和页面曝光不直接生成财务单据；已支付订单的优惠承担结果由订单、结算和财务模块承接。

## 2. 核心对象总览

| 对象组 | 核心对象 | 业务事实 | 主要责任模块 |
|---|---|---|---|
| 营销规则 | 营销活动、优惠券 | 在什么条件下可给客户或分销商什么优惠 | 营销中心 |
| 营销投放 | 发券批次、发券任务、分销商券额度 | 哪张券发给谁、何时发、发多少 | 营销中心 |
| 权益账户 | 会员优惠券、分销商优惠券 | 某会员或分销商实际拥有并可使用的券 | 客户、渠道、营销中心 |
| 订单承接 | 订单优惠明细、优惠承担明细 | 某订单实际享受什么优惠、各方各承担多少 | 销售中心、财务 |
| 小程序运营 | 页面、页面版本、页面模板、组件素材 | 哪个组织在什么时间使用什么页面呈现营销内容 | 营销中心 |
| 分析 | 营销归因记录 | 活动、券、页面与订单、优惠成本之间的关联 | 营销中心、数据中心 |

## 3. 必须区分的四类对象

### 3.1 营销活动

营销活动描述“满足条件后系统自动给予的优惠或权益”。例如提前30天预订立减、4人同行减、同舱第三人优惠、指定团期尾位特卖。

营销活动可以关联优惠券作为领取入口，但活动本身不是优惠券，不应要求每个参与者都先领券。

### 3.2 优惠券

优惠券描述“持券人可以在满足使用条件时享受的优惠权益”。例如满10,000减500、生日月券、签证服务券、分销商采购减免券。

优惠券定义长期稳定的优惠规则；其实际发放对象、时间、数量和本次限定范围由发券批次和发券任务承接。

### 3.3 发券任务

发券任务描述“一次具体投放”。例如 2026 年 7 月 15 日向华北黑钻会员直接发放 1,000 张邮轮券，或向指定分销商分配 100 张 C 端券额度。

发券任务只能在原优惠券规则允许的范围内收窄，不能扩大产品、团期、渠道、有效期、优惠金额或承担方范围。

### 3.4 小程序页面

小程序页面只负责展示和导流，例如首页、活动页、门店页和分销分享页。页面组件可以绑定活动、优惠券和产品，但不得在页面配置中修改优惠规则、价格、承担比例或销售授权。

## 4. 营销规则对象

### 4.1 营销活动

| 项目 | 定义 |
|---|---|
| 对象名称 | 营销活动 |
| 主键建议 | marketing_campaign_id，编号示例：MKT-ACT-20260712-001 |
| 上游 | 产品、方案、团期、自由行出行日期、服务项目、会员分群、门店、分销商、定价策略 |
| 下游 | 订单优惠明细、优惠承担明细、活动页、营销归因记录、审批单 |
| 责任模块 | 营销中心 |
| 财务触发 | 活动发布不触发财务；已支付订单使用活动优惠后，由订单和结算承接承担金额 |

营销活动核心字段：

- 活动名称、活动类型、负责人、活动主体组织。
- 预订时间、出发时间、活动生效时间和结束时间。
- 适用销售渠道、组织、门店、分销商或会员分群。
- 适用产品类型、产品、方案、团期、出行日期、舱型、铺位、服务项目和费用项。
- 触发条件：提前天数、人数、指定游客类型、会员等级、首单、复购、支付方式等。
- 优惠规则：直减、折扣、每人优惠、第 N 人优惠、多人阶梯、服务兑换。
- 合用规则、优惠优先级、优惠名额、预算和承担比例。
- 关联页面、投放素材和审批状态。

### 4.2 优惠券

| 项目 | 定义 |
|---|---|
| 对象名称 | 优惠券 |
| 主键建议 | coupon_id，编号示例：MKT-CPN-20260712-001 |
| 上游 | 营销运营配置、产品与团期、定价策略、承担规则 |
| 下游 | 发券批次、发券任务、会员优惠券、分销商优惠券、订单优惠明细 |
| 责任模块 | 营销中心 |
| 财务触发 | 券创建、停发、发放不触发财务；已支付订单使用后进入订单、结算和财务承接 |

优惠券核心字段：

- 券名称、对客展示名称、券类型、适用对象类型。
- 优惠金额、折扣、门槛、人数、游客类型、最高优惠金额和单订单上限。
- 可售渠道、产品范围、费用项范围、预订与出发日期范围。
- 领取时间、券生效时间、券失效时间或领取后有效天数。
- 单人限领、单人限用、总发放量、总优惠预算和预留名额。
- 与活动、其他券、会员价、积分抵扣、人工优惠的合用规则。
- 可承担主体和承担比例上限。

### 4.3 发券批次

| 项目 | 定义 |
|---|---|
| 对象名称 | 发券批次 |
| 主键建议 | coupon_batch_id，编号示例：MKT-CBN-20260712-001 |
| 上游 | 优惠券 |
| 下游 | 发券任务、会员优惠券、分销商优惠券、券使用统计 |
| 责任模块 | 营销中心 |
| 财务触发 | 不直接触发财务 |

发券批次固定一次投放的实际使用边界：本次有效期、投放总量、预算、承担比例、适用组织和产品范围。它只能缩小优惠券规则，不能放宽优惠券规则。

### 4.4 发券任务

| 项目 | 定义 |
|---|---|
| 对象名称 | 发券任务 |
| 主键建议 | coupon_issue_task_id，编号示例：MKT-ISS-20260712-001 |
| 上游 | 发券批次、会员分群、门店、分销商、名单导入 |
| 下游 | 会员优惠券、分销商优惠券、通知记录、发券结果 |
| 责任模块 | 营销中心 |
| 财务触发 | 不直接触发财务 |

发券任务核心字段：

- 接收对象类型：会员、指定客户、分销商、分销商代发 C 端券。
- 接收对象范围：分群、会员等级、门店归属、区域、分销商类型、授权产品、指定名单。
- 发放方式：直接到账、领取链接、领取码、分销商券额度。
- 发放时间：立即、定时；事件触发属于后续扩展，不与定时任务混为同一状态。
- 通知渠道：小程序账户、服务通知、短信、企业微信或分销商后台消息。
- 计划数量、成功数量、失败数量、重复过滤数量和取消数量。

## 5. 权益账户对象

### 5.1 会员优惠券

| 项目 | 定义 |
|---|---|
| 对象名称 | 会员优惠券 |
| 主键建议 | member_coupon_id，编号示例：MKT-MCP-20260712-001 |
| 上游 | 发券任务、会员自主领取、活动达成 |
| 下游 | 订单优惠明细、会员账户、券使用记录、退款返券记录 |
| 责任模块 | 客户、营销中心 |
| 财务触发 | 使用并形成已支付订单后，由订单和结算承接 |

会员优惠券必须绑定会员身份。未绑定会员的游客不能获得可跨订单使用的 C 端券；门店代客下单时仍需写入实际领券会员。

### 5.2 分销商优惠券

| 项目 | 定义 |
|---|---|
| 对象名称 | 分销商优惠券 |
| 主键建议 | distributor_coupon_id，编号示例：MKT-DCP-20260712-001 |
| 上游 | 发券任务、分销商激励活动 |
| 下游 | 分销订单优惠明细、分销结算、分销商账户 |
| 责任模块 | 渠道、营销中心 |
| 财务触发 | 使用并形成已支付分销订单后，由渠道结算承接 |

分销商优惠券分为两种，必须独立建账：

1. 分销商自用券：受益人是分销商，抵扣其应向凯撒支付的金额。
2. 分销商代发 C 端券：分销商仅取得可发放额度，最终券归属实际会员，订单保留分销商来源。

禁止将分销商自用券当作会员优惠券，也禁止把分销商代发额度直接当作分销商收入或佣金。

## 6. 订单承接对象

### 6.1 订单优惠明细

| 项目 | 定义 |
|---|---|
| 对象名称 | 订单优惠明细 |
| 主键建议 | order_discount_id，编号示例：MKT-DSC-20260712-001 |
| 上游 | 营销活动、会员优惠券、分销商优惠券、会员价、积分抵扣、人工优惠 |
| 下游 | 应收单、售后申请、退款单、结算单、营销分析 |
| 责任模块 | 销售中心 |
| 财务触发 | 已支付订单的应收、退款、结算和承担结果以该明细为依据 |

每一笔订单优惠明细必须保存：

- 原销售价、适用价格行、优惠计算基数和优惠后金额。
- 优惠来源类型：自动活动、优惠券、会员价、积分、人工优惠。
- 关联活动、优惠券、发券批次和发券任务。
- 受益主体：会员、客户、分销商或订单。
- 使用状态：已预留、已使用、已释放、已冲回、已返券。
- 订单行和游客维度的分摊结果。

### 6.2 优惠承担明细

| 项目 | 定义 |
|---|---|
| 对象名称 | 优惠承担明细 |
| 主键建议 | discount_share_id，编号示例：MKT-SHR-20260712-001 |
| 上游 | 订单优惠明细、活动或优惠券承担规则 |
| 下游 | 门店结算、分销结算、营销成本分析、结算单 |
| 责任模块 | 销售中心、财务 |
| 财务触发 | 订单结算或退款结算时形成实际承担金额；不在营销页面生成财务单据 |

承担主体只允许为：集团总部、区域公司、订单归属门店、指定门店、分销商、合作方、供应商营销补贴方。每笔订单优惠的承担金额合计必须等于该笔实际优惠金额。

## 7. 小程序运营对象

### 7.1 页面与页面版本

| 对象 | 定义 | 主键建议 | 上游 | 下游 | 责任模块 |
|---|---|---|---|---|---|
| 小程序页面 | 一个可被投放的首页、活动页、专题页、门店页或自定义页 | mini_page_id，MKT-PAGE-日期-序号 | 模板、组织范围、组件配置 | 页面版本、分享链接、曝光记录 | 营销中心 |
| 小程序页面版本 | 某页面在一次发布周期内的不可变配置快照 | mini_page_version_id，MKT-PVER-日期-序号 | 小程序页面、组件配置 | 预览、审核、发布、回滚 | 营销中心 |

页面范围采用继承关系：指定门店页面 > 区域页面 > 集团默认页面。分销商默认只能获得带渠道归属的分享页，不直接编辑集团或门店小程序页面。

### 7.2 页面模板与组件素材

| 对象 | 定义 | 主键建议 | 上游 | 下游 | 责任模块 |
|---|---|---|---|---|---|
| 页面模板 | 可复用的页面布局和组件结构，不直接对外生效 | mini_page_template_id，MKT-TPL-日期-序号 | 运营维护 | 页面草稿 | 营销中心 |
| 组件素材 | 可复用的图片、视频、海报、品牌素材或门店素材 | marketing_asset_id，MKT-AST-日期-序号 | 素材上传 | 页面组件、活动投放 | 营销中心 |

模板只生成页面草稿，不能直接覆盖使用中的线上页面。组件只能绑定已生效的产品、活动和优惠券，不得在组件属性中自行输入价格或优惠金额。

## 8. 适用范围模型

### 8.1 产品范围层级

营销规则按以下层级逐级收窄：

```text
全部可营销产品
-> 产品类型
-> 目的地/主题
-> 指定产品
-> 指定方案
-> 团期/航次/班期/出行日期
-> 舱型/铺位/游客价格行/服务费用项
```

规则不得跳过产品销售授权：门店、分销商或小程序只能使用其已被授权销售的产品和团期。

### 8.2 可适用产品类型

- 跟团游、邮轮、专列、研学：可精确到团期、航次、班期、舱型、铺位和游客价格行。
- 自由行：可精确到套餐、出行日期价格库存、房型和附加服务。
- 单项服务：可精确到服务项目、服务费或指定可营销服务项。
- MICE：默认不参与标准券和自动活动；如需优惠，必须关联锁定报价或专项审批。
- 外采：跟随原产品类型和渠道授权；不得因营销规则泄露采购价或绕过差额结算。

### 8.3 默认排除费用

以下费用默认不进入满减、折扣和按人优惠基数，只有专项规则才允许抵扣：

- 税费、港务费、燃油附加等刚性代收项。
- 保险代收、签证代垫、供应商全额代收款。
- 升舱、改舱、改铺、单房差等平进平出或损益需单独核算的项目。
- 已被明确为不可优惠的服务费或补差款。

## 9. 优惠计算与合用规则

### 9.1 优惠类型

营销活动和优惠券均可采用以下优惠计算方式：

- 满减：达到订单金额门槛后减固定金额。
- 直减：无门槛或指定条件下减固定金额。
- 折扣：按比例折扣，并设置最高优惠金额。
- 按人优惠：按成人、儿童、指定游客类型或可优惠人数计算。
- 第 N 人优惠：按订单内游客排序对指定游客优惠。
- 多人阶梯：按人数匹配最高满足阶梯。
- 服务兑换：兑换接送机、保险、签证服务、贵宾厅等非现金权益。

“早订、尾位、会员专享、银行合作、门店开业”是触发场景，不是独立金额计算对象；它们应选择上述某一种优惠计算方式。

### 9.2 默认计算顺序

除非活动明确配置互斥组，订单按以下顺序计算：

```text
已生效销售价
-> 自动营销活动
-> 会员价
-> 优惠券
-> 积分抵扣
-> 人工优惠或售后调整
```

默认券使用基数为“已参与自动活动和会员价后的可优惠金额、积分抵扣前金额”。任何改变默认计算顺序的规则必须在活动或优惠券中显式配置，并保留订单实际计算快照。

### 9.3 合用规则

每个活动和优惠券必须配置：

- 与其他金额类优惠券：不可叠加、可叠加或仅同互斥组取最大优惠。
- 与自动营销活动：可叠加、不可叠加或取最大优惠。
- 与会员价：可叠加或互斥。
- 与积分抵扣：可叠加或互斥。
- 与门店人工优惠：默认互斥；允许叠加时需审批。

单订单默认最多使用一张金额类优惠券；服务兑换券可按规则与金额类优惠券并存。

## 10. 名额、预算与使用生命周期

### 10.1 优惠名额

优惠名额分为活动名额、发券批次数量、会员/分销商限领数量和订单使用次数。四者不能互相替代。

### 10.2 预算占用

- 发券不占用实际优惠成本，只占用可发放数量。
- 下单预留优惠时，可预留活动名额和券使用资格。
- 已支付订单确认实际优惠金额和承担金额。
- 未支付超时、取消订单或资格不满足时，释放预留名额。
- 预算耗尽后，活动或券停止新的使用资格；已支付订单不受影响。

### 10.3 会员优惠券状态

```text
待生效 -> 可使用 -> 已预留 -> 已使用
                  -> 已过期
                  -> 已作废
已使用 -> 已冲回/已返券
```

### 10.4 发券任务状态

```text
草稿 -> 待执行 -> 发放中 -> 已完成
                         -> 部分失败
草稿/待执行 -> 已取消
```

### 10.5 活动、优惠券和页面状态

```text
草稿 -> 待审批 -> 待生效 -> 生效中/使用中 -> 已暂停或已停发 -> 已结束/已下线
```

状态命名必须服务对象本身：活动用“进行中”，优惠券用“生效中/已停发”，页面用“使用中/已下线”，不得混用。

## 11. 优惠承担、佣金与结算口径

### 11.1 承担规则

一笔已使用优惠必须拆成明确承担方金额，承担比例合计为 100%。承担规则可在活动、优惠券或发券批次中配置，订单生成后固化快照，不得被后续规则修改覆盖。

### 11.2 门店利润与佣金

门店佣金、门店资金池和分销佣金不能仅按“客户实付金额”统一计算，必须读取优惠承担明细：

- 总部承担优惠：不减少门店可结算利润。
- 区域承担优惠：由区域营销成本承担，不减少门店可结算利润。
- 门店承担优惠：从门店利润、资金池或门店应结中扣减。
- 分销商承担的 C 端优惠：从分销商应结金额或佣金中扣减。
- 分销商自用券：减少分销商应向凯撒支付的金额，不作为 C 端客户优惠。

佣金政策只定义佣金计算规则，营销域只向其提供“原订单金额、优惠金额、承担方金额、结算口径金额”等结果，不在营销页配置佣金点。

### 11.3 财务承接

营销页面不生成应收、退款、应付、付款、发票或 NC 凭证。订单应收按客户实际应付金额生成；优惠承担金额在订单结算、门店结算、分销结算和营销分析中承接。

## 12. 退款、改期与售后口径

### 12.1 未支付取消

订单取消或支付超时后：释放活动名额、优惠券预留和预算预留；优惠券恢复为可使用状态。

### 12.2 全额退款

全额退款按活动或券规则决定：不返券、返还原券、返还新券。返还原券时必须明确恢复原有效期或延长有效期，不能默认永久有效。

### 12.3 部分退款

部分退款按订单行和游客维度分摊优惠；退款后若订单不再满足满减、多人或第 N 人条件，必须通过售后调整重新计算优惠差额，不允许只按退款金额线性扣减。

### 12.4 改期、转团和改舱

改期、转团、改舱、改铺后必须重新校验活动、券、出发日期、舱型、铺位和人数条件。原优惠不自动继承到不满足条件的新订单对象；差额通过售后调整单承接。

## 13. 审批与权限边界

### 13.1 审批触发

以下情形必须进入审批：

- 优惠后低于最低销售价、渠道保护价或合同约定价。
- 全国范围活动、全国发券或集团首页发布。
- 使用集团或跨区域营销预算。
- 门店、分销商、供应商或合作方承担优惠金额。
- 多种优惠叠加后超过预设上限。
- MICE 锁定报价、已结算订单或特殊外采产品参与优惠。

### 13.2 权限边界

| 角色 | 可维护范围 |
|---|---|
| 总部营销运营 | 全国活动、优惠券、发券任务、集团与区域页面模板 |
| 区域营销 | 本区域活动、预算、发券任务和区域页面 |
| 门店经理 | 本门店承担的活动、指定会员发券、门店页面草稿 |
| 渠道经理 | 负责分销商的券、代发额度和推广页 |
| 产品运营/团控 | 确认可参与的产品、团期、库存和停售规则 |
| 财务 | 查看承担、结算与报表结果，不维护营销规则 |
| 审批人 | 审核规则、预算和发布，不替代规则对象维护 |

## 14. 业财事件承接

营销域事件不单独替代订单、收款、退款和结算事件。它们应按以下方式承接：

| 营销事件 | 来源对象 | 结果 | 对业财影响 |
|---|---|---|---|
| 活动/优惠券发布 | 营销活动、优惠券 | 规则生效 | 不直接生成财务单据 |
| 发券完成 | 发券任务 | 生成权益账户记录 | 不直接生成财务单据 |
| 订单优惠预留 | 销售订单、优惠券/活动 | 生成订单优惠预留 | 不生成应收，释放时可逆转 |
| 订单优惠确认 | 已支付销售订单 | 固化优惠和承担明细 | 应收、收款、结算按实际应付和承担结果承接 |
| 优惠冲回/返券 | 售后申请、退款单 | 生成冲回或返券记录 | 由应收冲减、退款和结算调整事件承接 |
| 结算承担确认 | 结算单 | 固化各方承担金额 | 影响门店、分销和营销成本报表 |

## 15. 禁止混用清单

1. 营销活动不等于优惠券，不要求自动活动先发券。
2. 优惠券不等于发券任务，券规则不得被每个发券任务随意改写。
3. 发券任务不等于会员分群，分群只提供接收对象范围。
4. 小程序页面不等于营销规则，页面只能展示和绑定已生效对象。
5. 分销商不是会员；分销商自用券与分销商代发 C 端券必须分账。
6. 优惠承担不等于佣金；佣金政策不得在营销中心重复配置。
7. 优惠后价格不等于基础销售价；基础销售价仍由定价策略维护。
8. 未支付订单预留不等于已使用优惠；退款、取消和超时必须可释放或冲回。
9. 优惠承担不直接等于财务费用；实际财务确认以订单、退款和结算事件为准。
10. OTA 平台促销默认不进入凯撒营销优惠引擎，除非后续建立专门的平台补贴和对账规则。

## 16. 第一阶段验收标准

1. 营销活动、优惠券、发券批次、发券任务、权益账户、订单优惠明细和承担明细均有独立定义。
2. C 端券、分销商自用券和分销商代发 C 端券的受益人、使用订单和结算口径明确。
3. 跟团、邮轮、专列、自由行、单项服务、MICE、外采产品的适用边界明确。
4. 优惠计算顺序、合用规则、名额与预算占用、退款返券、改期重算规则明确。
5. 总部、区域、门店、分销商、合作方和供应商营销补贴的承担口径明确。
6. 营销域不重复维护基础销售价、佣金规则、产品授权和财务单据。
7. 后续活动管理、优惠券、发券中心和小程序页面配置均能回到本文对象定义。
