# 二级页面通用 UI 规范

## 1. 适用范围

本规范适用于后台从列表页进入的二级详情、新建、编辑、审批详情和配置页面。

典型页面：

- 产品编辑页
- 产品详情页
- 团期新建页
- 团期详情页
- 订单详情页
- 供应商详情页
- 财务结算详情页

## 2. 顶部标题区

二级页标题区固定为：

`返回 + 当前页面标题 + 右侧操作区`

规则：

- 返回文案只显示 `返回`。
- 页面标题使用 `18px / 26px / 600`。
- 不在内容区重复面包屑。
- 右侧只放页面级主操作。
- 页面级主操作只出现一次，摘要卡、详情卡和表单卡内不重复放同一组主操作。

## 3. 宽度规则

二级页面分两层宽度：

- 页面画布、标题区、摘要卡、步骤 Tab 或详情 Tab：`100%`。
- Tab 下真实内容轨道：最大 `1216px`，左对齐。

例外：

- 宽表格需要横向承载完整业务列时，可在表格容器内滚动。
- 看板、财务报表、复杂分析页可按看板规范处理。

## 4. 编辑页与详情页差异

编辑页：

- 使用输入框、选择器、上传控件和可编辑明细卡。
- 一个 Tab 内尽量是一个整体表单卡。
- 底部步骤按钮跟随当前 Tab。

详情页：

- 使用只读字段、业务列表、附件预览、状态和跳转入口。
- 不把详情页做成表单输入感。
- 不展示说明性建模语言、字段口径、对象链路或系统内部来源。

## 5. Tab 规则

步骤 Tab 和详情 Tab：

- 宽度铺满页面内容区。
- 选中态可以使用浅蓝底和主色文字。
- 不使用粗下划线。
- active 不导致高度、字号、布局跳动。
- Tab 下内容进入 `1216px` 左对齐内容轨道。

## 6. 线条与颜色

默认视觉必须克制：

- 卡片边框使用中性灰。
- 分割线使用中性浅灰。
- 输入框普通态边框使用中性灰。
- 不使用明显蓝色边框作为默认边线。
- 蓝色只用于主按钮、链接、当前步骤、选中态、焦点态和必要的业务状态。

## 7. 摘要卡边界

摘要卡用于确认当前对象，不承担业务解释职责。

本规范暂不强制所有二级页摘要卡结构完全一致。产品编辑页和产品详情页可以保留各自编辑态、展示态差异，但信息层级必须克制，不放重复主操作和大段说明。

## 8. 验收清单

- 返回入口是否只显示 `返回`。
- 页面标题字号是否为 `18px / 26px / 600`。
- 页面级主操作是否只出现一次。
- Tab 是否 100% 铺满。
- Tab 下真实内容是否进入 `1216px` 左对齐轨道。
- 默认边线是否为中性灰。
- 蓝色是否只用于交互状态。
- 详情页是否避免表单输入感。
- 页面是否没有说明性建模语言。
