# 第 5.2 步：标准跟团/国内游/出境游链路验收报告

## 验收范围

本轮验收标准跟团、国内游、出境游主链路，不包含邮轮、专列、自由行、单项服务、研学和单团自组。

主链路：

产品中心 > 自营产品 > 产品管理 → 履约中心 > 团期管控 → 销售中心 > 订单管理 → 销售中心 > 合同管理 → 销售中心 > 收款认领 → 财务 > 应收管理 / 收款认款 → 履约中心 > 团期结算 → 财务 > 业务结算 → 财务 > NC推送。

## 验收方式

- 已完成源码级链路核对：列表入口、详情入口、跳转参数、抽屉口径、业财承接方向。
- 未完成人工浏览器点击：当前 in-app browser 不可用，因此本报告不表述为人工点击已通过。
- 不做真实跨页数据回写验证，只判断业务流程和页面承接是否符合原型目标。

## 总体结论

标准跟团/国内游/出境游主链路基本存在，普通订单口径没有被单团项目改坏。

已确认成立的点：

- 旅游线路列表支持出境游、国内游筛选，线路详情区分线路信息、行程方案、团期。
- 线路详情明确“参考起价仅作线路展示起价，具体销售价在团期层维护”，符合线路、方案、团期三层模型。
- 团期管控承接普通团期的日期、名额、渠道余量、销售状态、执行状态、成本状态。
- 普通订单仍使用普通订单视图，收款为定金、尾款、补差、团费等口径，不是项目阶段收款。
- 普通订单详情仍使用游客资料、证件/签证资料、旅游合同、出团确认单等口径。
- 财务侧存在普通订单应收、普通流水认款、普通团期成本、普通团期结算和普通 NC 推送。

但本轮发现 1 个 P0 链路错误和 2 个 P1 对象准确性问题，需要下一轮优先整改。

## 发现的问题

### P0-001 财务 > 收款认款：普通认款行反查链接跳到单团项目

页面：`merchant/finance/finance-matching.html`

问题：

- 第一条普通流水认款是 `KS2024061801 / 欧洲十国经典游 / SK20240618001`。
- 行操作里却出现 `查看订单` 链接到 `orders-detail.html?type=project&source=customProject&orderNo=KSMICE20260704001...`。
- 同一行还有 `查看项目` 链接到 `product-custom-detail.html?projectNo=MICE-20250623-001`。

影响：

- 财务从普通收款认款反查，会直接跳到单团项目订单和单团项目详情。
- 这是对象类型错误，不是普通体验问题。

建议整改：

- 普通认款行只保留普通订单反查：`sales/orders-detail.html?type=group&orderNo=KS2024061801&scheduleNo=EU-FRA-20250720-001`。
- 单团项目反查链接只能在 `data-claim-mode="projectStage"` 或 `data-order-type="单团项目"` 的行显示。

### P1-001 产品中心 > 自营产品 > 产品管理：不同线路详情未带线路参数

页面：`merchant/product/products.html`

问题：

- 欧洲十国、日本关西、三亚亲子等多行详情链接都是 `products-detail.html`。
- 当前 `products-detail.html` 默认展示欧洲十国经典游。

影响：

- 从日本关西或三亚亲子点击详情，容易进入欧洲十国详情。
- 这会影响从线路源头验证国内游、出境游不同线路链路。

建议整改：

- 列表详情链接补 `routeCode`、`productName`、`type`。
- 线路详情页按参数切换标题、类型、方案和团期列表。

### P1-002 线路详情 > 团期 Tab：团期行详情未带团期编号

页面：`merchant/product/products-detail.html`

问题：

- 团期 Tab 内每行“详情”都是 `../tour/schedules-detail.html`，没有 `scheduleNo`。
- 团期详情页支持 `scheduleNo` 入参，但这里没有传。

影响：

- 从线路详情里点不同团期，可能都进入默认团期详情。

建议整改：

- 每行团期详情链接补 `scheduleNo`、`product`、`package`、`type=group`。
- 例如：`../tour/schedules-detail.html?type=group&scheduleNo=EU20240728001&product=欧洲十国经典游&package=A线 法德意12天`。

### P2-001 财务 > 应收管理：普通订单应收反查订单入口偏弱

页面：`merchant/finance/finance-receivable.html`

问题：

- 普通订单应收行主要只有“详情”，单团项目行有“查看订单/查看项目/查看项目团/查看认款”。

影响：

- 财务核普通订单应收时，反查订单要多一步，不如项目订单清楚。

建议整改：

- 普通订单应收行增加 `查看订单`。
- 若能识别团期，补 `查看团期`。
- 项目反查入口只在项目应收行显示。

## 本轮不判定为问题的点

- 订单列表普通行静态 href 没带 `orderNo`：脚本点击时会通过 `detailHrefFromRow(row, href)` 补齐 `orderNo` 和订单上下文，运行态可承接。
- 团期管控隐藏表格静态 href 没带 `scheduleNo`：运行态列表通过 `detailHref(item)` 生成带参数的团期详情链接。
- 普通订单保留普通订单录入入口：这是标准跟团/国内游/出境游正常业务入口，不属于单团项目问题。
