本文用于固化机票切位/包机、邮轮包舱、专列包铺和散票 BSP 的完整业务方案。后续涉及交通采购资源、团期资源分配、出票确认、票损/舱损/铺损、付款节点、团期成本和 NC 推送的页面改造，均以本文为专项基线。

## 1. 总体定位

交通批量采购资源不是航空、邮轮、专列的基础档案，而是围绕具体日期、具体数量、具体成本、DL 截止日、分期付款和团期占用形成的动态采购履约对象。

业务主链路：

```text
资源主数据 -> 交通采购合同 -> 锁定库存 -> 团期分配 -> 订单消耗 -> 出票/确认 -> 损耗核算 -> 团期成本 -> 财务付款/NC
```

模块边界：

| 模块 | 承载内容 | 不承载内容 |
|---|---|---|
| 资源 | 航司、航线、航司协议、船公司、船只、邮轮模板、专列运营商、车次/铺位模板、供应商档案 | 具体日期库存、团期分配、出票、损耗、付款执行 |
| 履约中心 / 交通采购 | 交通采购合同、动态库存、团期分配、释放、还位、出票、损耗确认入口 | 财务付款执行、NC 真实推送 |
| 团期成本 | 实际交通成本、票损/舱损/铺损、退票冲减、成本确认 | 采购合同主维护、付款执行 |
| 财务 | 应付、付款申请承接、付款执行、回单、NC 凭证、BSP 对账付款 | 业务库存分配、团期调剂 |
| 审批 | 付款审批、超 DL 还位、异常损耗确认、成本确认审批 | 业务对象主数据维护 |

主入口建议放在：

```text
履约中心 / 交通采购 / 机票切位、邮轮包舱、专列包铺
```

现有菜单已调整为独立二级菜单“交通采购”。资源模块只提供基础资料和协议规则来源，不作为日常操作入口；团期管控保留交通资源 Tab 作为团期反查和处理入口。

## 2. 适用范围

### 2.1 纳入统一框架

| 资源类型 | 采购模式 | 核心库存单位 | 典型业务 |
|---|---|---|---|
| 机票 | 包机 | 座位，可按舱位拆分 | 旺季热门航线整机采购 |
| 机票 | 切位/系列位 | 座位 | 固定航班、固定日期或系列日期的团队位 |
| 邮轮 | 包舱 | 舱室，按舱型分层 | 航次内舱、海景、阳台、套房库存 |
| 专列 | 包铺/包厢 | 铺位或包厢，按席别和铺位分层 | 软卧、硬卧、上中下铺、整包厢 |

这四类共用“交通采购合同 + 库存子类型 + 付款节点 + 团期分配 + 实际使用 + 损耗核算”的框架。

### 2.2 不纳入统一库存池

散票采购不进入交通批量采购库存池。散票按订单或游客形成出票记录，后续通过 BSP 账单或 GDS/BSP 对账进入财务付款。

邮轮岸上游、专列地面配套、酒店、餐饮、景区、地接、接送机等，不进入交通主体库存池。这些费用走团期成本中的地接/供应商费用逻辑，在团期成本和结算单中与交通成本汇总。

## 3. 核心角色和高频任务

| 角色 | 高频任务 | 关注字段 | 下游影响 |
|---|---|---|---|
| 产品/采购 | 根据季度或年度计划签订包机、切位、包舱、包铺协议 | 供应商、合同号、资源类型、日期、数量、价格、付款节点、DL | 形成动态交通库存和成本基准 |
| 计调/团控 | 把交通库存分配给团期，销售中追加、释放、调剂，DL 前决策还位 | 可分配量、团期占用、已售、未售、DL、释放规则 | 影响团期可售容量、订单库存、票损 |
| 航服/票务 | 提交名单，出票或确认舱/铺，录入 PNR、票号、确认号 | 游客名单、证件、PNR、票号、出票状态 | 影响实际成本和损耗核算 |
| 销售顾问 | 销售团期产品，订单报名占用团期资源 | 团期可售余量、库存风险 | 影响团期已售数量，不直接操作采购合同 |
| 财务 | 审核付款、执行付款、上传回单、推送 NC、处理 BSP 对账 | 付款申请号、付款节点、应付金额、回单、NC 状态 | 影响付款状态、预付款核销和财务凭证 |
| 审批人 | 审批付款、超 DL 还位、异常损耗和成本确认 | 合同、金额、原因、团期影响 | 控制业务动作生效 |

## 4. 业务对象模型

交通批量采购资源由以下对象组成：

| 对象 | 说明 | 关键字段 |
|---|---|---|
| 交通采购合同 | 包机、切位、包舱、包铺的履约主对象 | 资源类型、采购模式、供应商、合同号、资源名称、日期范围、DL 日、保底数、免损数、合同金额、币种 |
| 库存子类型 | 按资源差异拆分库存和价格颗粒度 | 舱位等级、舱型、铺位类型、数量、单价、单房差、包厢标记 |
| 付款节点 | 一压、二压、三压/尾款等分期付款计划 | 节点名称、应付日期、比例/金额、状态、付款申请号、回单、NC 状态 |
| 团期分配记录 | 某合同资源分配给某团期的数量和成本 | 团期编号、线路、出发日期、子类型、分配数量、分配单价 |
| 订单占用记录 | 订单游客对团期已分配资源的销售消耗 | 订单号、游客、团期、子类型、占用状态 |
| 实际使用记录 | 出发前锁定的出票、出舱、出铺结果 | 游客、PNR、票号、舱号、铺位号、实际成本、出票状态 |
| 释放/还位记录 | 团期释放回库或向供应商还位 | 来源团期、子类型、数量、动作类型、是否免损、原因 |
| 损耗成本记录 | 票损、舱损、铺损的成本归属 | 损耗类型、子类型、损耗数量、比例、金额、归属团期、成本单号 |

## 5. 三种机票采购模式

### 5.1 包机

包机是旅行社向航空公司租用整架飞机或近似整机座位资源。系统按交通采购合同管理，可按舱位等级拆分库存子类型。

关键规则：

1. 旅行社承担整机销售风险。
2. 可自主分配给多个产品线和团期。
3. 合同金额和付款节点通常较重，付款风险需提前预警。
4. 未售座位按合同规则形成票损或整体损耗成本。

### 5.2 切位/系列位

切位是本方案第一优先级。旅行社与航司在固定航班、固定日期或系列日期上约定团队座位、单价、付款节点、免损和 DL。

完整流程：

```text
需求规划 -> 协议签署 -> 创建系列位合同 -> 一压付款 -> 分配团期 -> 订单销售消耗 -> 二压付款 -> DL 还位决策 -> 出票 -> 三压/尾款 -> 票损确认 -> 团期成本 -> 财务核销/NC
```

关键规则：

1. 系列位合同保存航司、合同号、航班号、舱位等级、座位数、单价、合同金额、付款节点、免损名额、DL 日和票损规则。
2. 一压支付后资源正式锁定，采购合同状态进入履约中。
3. 团期从系列位库分配座位，形成团期分配记录。
4. 订单报名只消耗团期已分配座位，不直接修改系列位合同。
5. 销售不足时先释放回交通资源库，DL 前可申请还位给航司。
6. 出票后按实际出票量与保底量核对票损。
7. 票损必须归属到具体团期成本，不允许混入合同层无法追溯。

### 5.3 散票

散票通过 GDS 或 BSP 按需出票，不承诺批量库存。

处理方式：

1. 散票不进入交通采购合同库存池。
2. 订单或游客形成出票记录，记录票号、PNR、实际票价、退改签费用。
3. 财务定期导入 BSP 账单，与出票记录核对数量、价格、退改签费用。
4. 差异项人工确认后生成付款指令和 NC 凭证。

## 6. 邮轮和专列共用框架

机票、邮轮、专列的底层流程一致：

```text
签订批量采购协议 -> 锁定资源库存 -> 按节点分期支付 -> 分配给团期 -> 销售阶段动态管理 -> 出发前确认实际用量 -> 按实际用量结算尾款 -> 未使用资源形成损耗
```

差异通过资源类型和库存子类型配置处理。

| 差异点 | 机票 | 邮轮 | 专列 | 处理方式 |
|---|---|---|---|---|
| 库存单位 | 座位 | 舱室，按舱型 | 铺位或包厢，按席别/铺位 | 库存子类型 |
| 定价结构 | 单一舱位价或少量舱位价 | 多舱型价格、单房差 | 多铺位价格、包厢价 | 子类型价格矩阵 |
| 损耗口径 | 票损 | 舱损 | 铺损 | 子类型损耗规则 |
| DL 严格程度 | 通常出发前 15-30 天 | Final Payment Date 通常更早 | 按合同灵活配置 | 协议级 DL 和预警规则 |

邮轮规则：

1. 舱型库存独立管理，内舱、海景、阳台、套房不可混算。
2. 单人占双人间时，库存按一间舱室消耗，收入侧收取单房差，成本侧仍按舱室成本。
3. 第三/第四人不新增舱室数量，但占用同一舱室容量，必须受最大入住人数约束。
4. 儿童占位按所选舱型计价并消耗舱室容量；儿童不占位使用独立儿童不占位价，不消耗舱室库存。
5. Final Payment Date 后未使用舱室通常按高比例或全额舱损处理。

专列规则：

1. 铺位按软卧、硬卧、上中下铺或包厢拆分。
2. 包厢模式按整包厢作为库存单位，不再独立销售内部铺位。
3. 成人和儿童占位必须选择席别和铺位，并消耗对应库存。
4. 儿童不占位使用独立儿童不占位价，不选择具体铺位，也不扣减铺位库存。
5. 铺损按席别、铺位或包厢规则独立计算。

### 6.1 邮轮/专列价格矩阵与占位规则

邮轮销售价格矩阵：

| 舱型 | 成人价 | 儿童占位价 | 儿童不占位价 | 第三/第四人价 | 单房差 |
|---|---:|---:|---:|---:|---:|

专列销售价格矩阵：

| 席别 | 铺位 | 成人价 | 儿童占位价 | 儿童不占位价 |
|---|---|---:|---:|---:|

库存扣减规则：

1. 成人默认占位，必须扣减所选舱型、席别或铺位库存。
2. 儿童占位按所选舱型、席别或铺位扣减库存。
3. 儿童不占位只生成订单价格行，不扣减舱室、铺位或包厢库存。
4. 邮轮第三/第四人不新增舱室数，但占用舱室容量并按第三/第四人价计收。
5. 售后退订时，儿童不占位只处理金额，不触发库存释放；儿童占位、成人和整舱/整铺退订才判断释放或损耗。

## 7. 关键数量口径

| 指标 | 计算口径 |
|---|---|
| 合同有效数量 | 原始采购数量 - 已无效/已取消数量 |
| 可分配数量 | 合同有效数量 - 已分配数量 - 已还供应商数量 - 已确认损耗数量 |
| 团期可售数量 | 本团分配数量 - 已售/已锁定游客数量 |
| 可释放回库数量 | 本团分配数量 - 已售数量 - 已出票/确认数量 |
| 实际交通成本 | 实际出票/确认数量 x 对应子类型成本单价 |
| 损耗数量 | 保底数量 - 实际使用数量 - 免损数量 - 已确认无损还位数量 |
| 损耗金额 | 损耗数量 x 子类型单价 x 损耗比例 |
| 邮轮儿童不占位数量 | 订单游客中儿童不占位人数，只影响收入和服务成本，不计入舱室库存消耗 |
| 专列儿童不占位数量 | 订单游客中儿童不占位人数，只影响收入和服务成本，不计入铺位库存消耗 |

邮轮和专列必须按子类型分别计算损耗，不能用统一平均价跨类型混算。

## 8. 状态设计

### 8.1 采购合同状态

```text
草稿 -> 待一压 -> 一压已付 -> 履约中 -> 临近DL -> 待出票/确认 -> 待三压 -> 待损耗确认 -> 结算中 -> 履约完成
```

异常状态：

- 已终止
- 已取消
- 付款逾期
- 还位异常
- 损耗争议

### 8.2 付款节点状态

```text
未到期 -> 待申请 -> 审批中 -> 待付款 -> 已付款 -> 已回单 -> 已推NC
```

异常状态：

- 已逾期
- 审批驳回
- 回单待补
- NC 推送失败

### 8.3 库存状态

```text
可分配 -> 已分配 -> 已售 -> 已出票/确认
```

库存动作状态：

- 已释放回库
- 已还供应商
- 已损耗
- 已退票/退舱/退铺

### 8.4 团期交通资源状态

- 未分配
- 已预占
- 库存不足
- 销售不足
- 临近 DL
- 名单待提交
- 已出票/已确认
- 待损耗确认
- 成本已确认

## 9. 页面规划

### 9.1 交通资源采购列表

使用角色：采购、计调、团控、航服、资源调度。

高频任务：

1. 看哪些合同临近付款。
2. 看哪些合同临近 DL。
3. 看哪些库存可分配、已分配、销售不足。
4. 快速进入分配、还位、付款申请和详情核对。

列表字段：

| 字段 | 说明 |
|---|---|
| 出发资源 | 机票显示出发航班，邮轮显示出发航次，专列显示去程车次 |
| 回程资源 | 机票显示回程航班，邮轮显示返港航次，专列显示返程车次；未采购时显示未采购 |
| 出发日期 | 出发资源对应日期，辅助展示 DL |
| 回程日期 | 回程资源对应出发日期；单程未采购时不硬填日期 |
| 用量 | 已用/总量、进度条、剩余量 |
| 待办事项 | 付款核实、DL还位核对、损耗确认、名单确认；每行只展示当前最需要处理的一类任务 |
| 操作 | 查看详情、分配资源 |

页面规则：

1. 不放 KPI 大卡片。
2. 不展示内部建模说明。
3. 不在列表中平铺复杂子流程入口。
4. 行操作只保留快速处理和进入详情。
5. 不把资源类型、采购模式、关联协议号作为拆分菜单后的首屏列展示；这些信息进入详情核对。

### 9.2 新建/编辑交通采购资源

承载形式：右抽屉。

表单分组：

1. 基础信息：资源类型、采购模式、供应商、合同号、协议日期。
2. 资源信息：航班、航次、列车、出发/到达、日期。
3. 库存子类型：舱位、舱型、铺位、包厢、数量、单价。
4. DL 与免损：DL 日、免损数量、保底数量、损耗规则。
5. 付款节点：一压、二压、三压/尾款，应付日期、比例或金额。
6. 附件：合同扫描件、供应商确认函。

字段展示规则：

- 机票：航班号、出发/到达机场、舱位等级、座位数、含税单价、名单截止日、出票截止日。
- 邮轮：船公司、船只、航次、出发港、返程港、舱型、舱室数、每间定员、单房差。
- 专列：运营商、列车号、出发/到达站、车厢、席别、铺位、包厢标记、铺位价或包厢价。

邮轮库存子类型至少支持：

- 内舱
- 海景
- 阳台
- 套房

专列库存子类型至少支持：

- 软卧上铺
- 软卧下铺
- 硬卧上铺
- 硬卧中铺
- 硬卧下铺
- 包厢

### 9.3 交通采购资源详情

详情页 Tab：

1. 合同信息
2. 库存与价格
3. 团期分配
4. 名单与出票
5. 付款节点
6. 损耗与成本

详情页原则：

1. 合同信息用于核对供应商、合同、DL、免损、付款规则。
2. 库存与价格用于查看子类型总量、单价、已分配、可分配。
3. 团期分配是追加分配、释放回库、跨团调剂的主入口。
4. 名单与出票承接游客名单、PNR/票号/舱号/铺位号。
5. 付款节点只发起和查看付款申请，不直接执行付款。

## 11. 特殊场景

### 11.1 跨团拆分

同一合同资源可以分配给多个团期。实际使用记录必须关联游客和团期，成本按团期拆分，不允许不同团期成本混算。

### 11.2 合并汇款

一笔付款可以关联多个合同或多个团期。财务付款单保存总额，业务侧保存拆分映射，NC 和团期成本按合同/团期维度追溯。

### 11.3 退团退票

订单售后发起退团后，根据是否出票、是否过 DL、供应商退改规则判断：

1. 可退：退票款或退舱/退铺金额冲减团期交通成本。
2. 不可退：形成票损/舱损/铺损，进入团期实际成本。
3. 对客退款时，售后流程按退改规则扣除对应损耗赔偿。

### 11.4 提前释放

释放回资源库和还位给供应商必须区分：

- 释放回资源库：团期不再占用，但合同资源仍可分配给其他团期。
- 还位给供应商：从合同资源退出，需判断是否免损或产生违约/损耗。

### 11.5 超卖补位

团期已售接近分配上限时预警。只能从合同可分配量追加；若合同无余量，可提示散票补位、其他合同调剂或关闭销售。

### 11.6 BSP 散票对账

BSP 账单导入后与出票台账比对：

- 票号是否存在
- 票价是否一致
- 税费是否一致
- 退改签费用是否一致
- 是否重复结算

差异项由财务人工确认后生成付款和 NC 凭证。

1. 交通采购资源的日常主入口在 `履约中心 > 交通采购`，不放回资源模块，也不再作为团期管控的子入口。

6. 审批中心只处理付款、超DL还位和异常损耗确认，不直接执行付款、成本入账或NC推送。
7. 机票、邮轮、专列共用交通采购资源框架；邮轮岸上游、专列沿途地接等地面配套仍走普通团期成本供应商费用。
8. BSP散票不进入批量采购库存池，只走出票台账、BSP对账、付款和NC。

### 28.1 修正后的核心结论

1. 机票采购协议支持单程、去程、返程和联程，不强制按往返一条协议建档。
2. 去程和返程建议作为两条独立协议记录，通过关联协议组绑定；单程采购可不绑定返程。
3. 系统到付款节点前只做提醒，不自动生成付款申请，也不自动进入审批流。
4. 一压后采购数量可变，但不能直接改原协议；必须创建交通采购变更单。
5. 已付款节点要按团期占用数量分摊到团期应付，团期成本必须显示总应付、已预付、待付。
6. 转团、团期取消或客人迁移时，机票预付分摊要生成调整单，从原团期转入新团期。
7. 团期结算必须同时校验应收和应付完整性；缺少应付项时不允许提交结算审核。

### 28.2 实际改进步骤

第一步：调整交通采购协议对象。

- 在交通采购协议中增加采购方向：单程、去程、返程、联程。
- 增加关联协议组号和关联协议号。
- 列表、详情和团期分配中展示协议方向，避免把去程和返程混成一个不可拆的文本资源。
- 去程和返程可以各自维护航班号、DL、数量、价格和付款节点。

第二步：调整付款节点动作。

- 付款节点到期前 N 天只推送提醒，默认提醒节奏为 7 天、3 天、1 天。
- 提醒内容包含协议号、方向、航班号、到期日、原协议数量、当前有效数量、建议应付金额、抵扣金额和本次建议付款金额。
- 经办人核实后手动点击发起付款申请。
- 系统不得自动生成付款申请草稿，不得自动进入审批流。

第三步：增加交通采购变更单。

- 变更单记录变更前数量、变更后数量、变更原因、影响节点和金额重算结果。
- 变更单走审批，审批通过后生成当前有效版本。
- 原协议历史版本保留，财务付款按最新有效版本计算。
- 一压多付金额自动抵扣二压或后续节点，在付款申请中明确标注。

第四步：建立机票预付分摊。

- 每个付款节点完成付款后，系统按团期占用数量比例计算预付分摊。
- 分摊公式：节点已付金额 x 团期分配数量 / 当前有效采购数量。
- 团期成本交通应付行展示总应付、已预付、调整金额、待付金额。
- 已预付部分不重复生成应付或付款申请。

第五步：支持转团预付分摊迁移。

- 客人从 A 团转到 B 团时，生成交通预付分摊调整单。
- A 团减少对应座位的预付分摊，B 团增加对应金额。
- 调整单记录原团期、新团期、订单、游客、协议号、金额、原因和操作人。
- 跨期时由 NC 按调整单生成调整凭证。

第六步：补齐团期应付完整性。

- 团期成本必须同时维护应收和应付。
- 应付来源包括机票预付分摊、机票待付差额、地接款、导游费、签证费和其他费用。
- 提交结算审核前校验交通应付、地接应付、导游/领队费用、预付分摊和转团调整是否完整。
- 缺失时只提示经办人补齐，不自动生成财务单据。
