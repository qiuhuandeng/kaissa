# 凯撒优选小程序 P0 页面清单与跳转关系

## 1. P0 目标

P0 版本只验证一件事：C 端消费者能从首页被种草，进入产品详情，完成日期、方案、出发地、人数选择，并顺利下单付款，最后在我的旅程中看到订单服务入口。

P0 不追求完整会员体系、完整 AI 能力、完整售后闭环和所有个人中心能力；这些入口可以先有，但不抢主链路。

## 2. 核心购买模型

小程序的旅游 SKU 不是单一商品，而是二维决策：

- 第一维：什么时候走，即团期/出发日期
- 第二维：怎么走，即方案，包含行程内容、酒店配置、服务标准和价格

下单锁定口径：

`团期ID + 方案ID + 出发地 + 人数`

后续所有页面都围绕这个口径承接，不在下单旅程中重新选择核心 SKU。

| 对象 | 用户看到的表达 | 系统承接含义 | P0 必须体现 |
| --- | --- | --- | --- |
| 线路 | 欧洲经典12日、北欧四国+冰岛 | 商品内容和种草主体 | 是 |
| 团期 | 4月11日出发、剩余12席 | 可售日期和库存主体 | 是 |
| 方案 | 经济/舒适/豪华 | 配置、服务标准和价格主体 | 是 |
| 出发地 | 北京/上海/广州 | 出发城市和价格差异 | 是 |
| 人数 | 成人/儿童/老人 | 价格计算和名额占用 | 是 |
| 订单 | 已预订的出行服务 | 用户付款与履约服务主体 | 是 |

## 3. P0 主链路

```text
首页
  -> 产品列表
  -> 产品详情
  -> 选日期·查方案半屏层
  -> 填写出行人
  -> 附加服务
  -> 确认订单
  -> 支付
  -> 下单成功
  -> 我的旅程/订单详情
```

底部 Tab P0 建议保留 5 个：

`首页 / 探索 / 我的旅程 / AI助手 / 我的`

其中 AI助手、我的在 P0 以轻量入口为主，不抢下单主链路。

## 5. 页面设计顺序

### 第一步：首页

目标：确定小程序视觉基准和转化入口。

必须包含：

- 多主体皮肤 Banner
- 智能搜索入口
- 目的地快速入口
- 精选旅游主题
- 出境热门
- 好看世界线路推荐
- 底部 Tab

验收重点：

- 首屏是否能明确传达凯撒优选的品牌质感
- 用户是否能在 3 秒内找到搜索、目的地、热门线路入口
- 首页模块是否都能自然导向产品列表或产品详情

### 第二步：产品列表

目标：承接所有首页入口和搜索结果。

必须包含：

- 搜索框
- 筛选条件
- 排序
- 已选筛选标签
- 产品卡片

产品卡片必须展示：

- 封面图
- 线路名称
- 核心卖点
- 方案类型
- 起步价和对应方案
- 最近可出发日
- 剩余名额提示
- 已售人数

### 第三步：产品详情

目标：先种草，再进入购买决策。

页面原则：

- 详情页展示线路级内容
- 不在详情主体中强行完成日期和方案选择
- 主按钮统一为“选日期·查方案”
- “咨询AI助手”始终可见，降低决策焦虑

必须包含：

- 头部媒体区
- 线路基础信息
- 行程 Tab
- 费用 Tab
- 评价 Tab
- 问答 Tab
- 底部固定操作栏

### 第四步：选购半屏层

目标：完成 P0 最关键的 SKU 决策。

交互顺序固定为：

1. 选出发日期
2. 选该日期下可售方案
3. 选出发地与人数
4. 查看费用合计
5. 立即预订

关键规则：

- 日期不可用时置灰
- 某日期没有某方案时隐藏该方案
- 出发地/人数变动时费用合计即时变化
- 面板可收起，用户可随时回看详情

### 第五步：下单旅程

目标：把已选 SKU 转为订单。

下单旅程不再重新选择团期和方案，只做填写与确认。

步骤：

1. 填写出行人
2. 附加服务
3. 确认订单
4. 支付
5. 下单成功

### 第六步：我的旅程

目标：承接付款后的服务触点。

必须包含：

- 订单列表
- 订单详情
- 付款信息
- 合同与出行材料
- 退款申请入口
- 联系客服入口
- 回团评价入口

## 6. P1/P2 暂缓范围

P1 再做：

- 真实 AI 对话能力
- 价格变动通知
- 新团期上线通知
- 完整优惠券中心
- 完整会员中心
- 完整发票申请
- 完整消息中心
- 退款审批进度细化

P2 再做：

- 多主体独立皮肤后台配置联动
- 个性化推荐算法
- 短视频自动播放
- 复杂签证规则引擎
- 出行中实时服务
- 回国后复购推荐

## 7. 后续每页设计前必须回答的问题

每开始一个页面前，先回答：

1. 这个页面处在购买旅程的哪一步？
2. 用户进入页面时最想解决什么问题？
3. 这个页面必须承接哪些上游参数？
4. 页面完成后要把用户送到哪里？
5. 哪个按钮是唯一主按钮？
6. 哪些能力先做入口，不在 P0 深做？

## 8. 下一步执行建议

下一步先做首页产品稿定稿，不继续扩页面数量。

首页定稿内容：

- 首页模块顺序
- 首屏视觉结构
- Banner 与搜索的关系
- 目的地入口数量和分组方式
- 精选主题与出境热门是否同时保留
- “好看世界”线路卡片信息字段
- 底部 Tab 视觉和命名

首页产品稿见：[miniapp-home-product-spec.md](miniapp-home-product-spec.md)。

首页定稿后，再进入产品列表设计。
